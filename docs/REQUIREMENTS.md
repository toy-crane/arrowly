# Arrowly — 요구사항 문서

> 화면 위에 자유롭게 그리는 macOS 오버레이 앱 (스크린 브러쉬 계열).
> 발표 · 화면녹화 · 강의 시 화면에 직접 낙서하는 용도.

## 결정된 것 (기술 스택)

- **프레임워크: Tauri** (Rust 백엔드 + OS 내장 웹뷰)
- **프론트엔드: Vite + React + TypeScript**
- **패키지 매니저: Bun** (프론트엔드 툴링 용도)
- **타깃: macOS** (Tauri라 추후 Windows/Linux 확장은 가능)

## ⚠️ 이 스타일 앱을 만들 때 조심할 것 (핵심)

기능보다 먼저, 이 오버레이 유틸리티 특유의 지뢰들. 대부분 "안 되는 게 아니라, 모르면 시간 날리는" 항목.

### 1. 투명 오버레이 창 (이 앱의 심장)
- 창을 **transparent + decorations 없음 + always-on-top + 전체 화면 덮기**로 구성
- **클릭 통과 토글이 핵심**: 그리기 OFF일 때는 밑 앱으로 클릭이 통과(`set_ignore_cursor_events(true)`), ON일 때만 이벤트 가로채기
- **전체화면 앱 위에도** 떠야 함 → 창 레벨/`visible-on-all-workspaces` 설정 확인
- **멀티 모니터**: 디스플레이마다 오버레이 창을 따로 생성해야 함

### 2. Tauri 특유
- 백엔드(메인 프로세스에 해당)는 **Rust** → 새로 배우는 영역. OS 밀착 기능(트레이·단축키·창제어)이 Rust 쪽에 삶
- UI(React) ↔ Rust는 **`invoke` / 이벤트(emit·listen)** 로 통신 (Electron의 IPC에 해당)
- 웹뷰가 **OS 내장 WebKit** → 크로미움 전용 CSS/API 가정 금지, macOS 웹뷰 기준으로 확인
- 창 제어는 `@tauri-apps/plugin-window` 등 **플러그인 권한(capabilities)** 을 명시적으로 허용해야 동작

### 3. macOS 권한 (TCC) — 프레임워크 무관하게 필요
- **손쉬운 사용(Accessibility)**: 전역 단축키·입력 감지에 필요할 수 있음
- **화면 기록(Screen Recording)**: 화면 캡처/저장 기능 넣으면 필요
- 코드로 조용히 못 켬 → **사용자가 시스템 설정에서 직접 켜야 하고, 그 안내 UX를 직접 만들어야 함**
- 개발 중 TCC가 꼬이면 "권한 줬는데 안 먹는" 현상으로 시간 소모 → 앱 재서명/재설치로 초기화

### 4. 배포 · 서명 · 공증 — 다 만들고 나서 만나는 벽
- **Apple Developer Program 연 $99** (배포/공증하려면 사실상 필수)
- **코드 서명 + 공증(Notarization)** 안 하면 남의 맥에서 Gatekeeper가 차단
- 배포 경로: **직접 배포(DMG)** 가 이런 오버레이 앱엔 일반적 (App Store는 심사·entitlement 이슈)
- **Apple Silicon / Intel** 아키텍처별 빌드 확인 (universal binary)

### 5. 맥 유틸리티 앱 관용 패턴 (꼭 지킬 것)
- **Dock 숨김 상주 앱**: Dock/앱 스위처에 안 뜨고 메뉴바에만 상주
  - ⚠️ Dock에서 숨기면 종료 수단이 사라짐 → **메뉴바에 "종료(Quit)" 항목 반드시 제공**
- **메뉴바 아이콘은 템플릿 이미지로** → 다크/라이트 모드 자동 대응 (안 하면 아이콘 안 보임)
- **로그인 시 자동 실행** 옵션 (상주 유틸의 기본 기대치)
- **전역 단축키**로 오버레이 켜기/끄기

### 6. 상태 · 데이터
- **멀티 윈도우 간 공유 상태는 Rust(백엔드) 쪽에서 관리** — 각 창의 React 트리는 서로 독립
- 로컬 저장은 파일/`tauri-plugin-store` 등 (서버 DB 없음)

### 7. 그리기 엔진
- Canvas 기반. **점을 그냥 이으면 각져 보임** → 곡선 보간(smoothing) 필요
- 그림이 많아지면 리렌더 성능 고려

## 학습 · 구현 추천 순서

1. Tauri + Vite/React 프로젝트 부트스트랩 (Bun)
2. **투명 오버레이 창 띄우기 (제일 먼저 성공시킬 실험)**
3. 그 위에 마우스로 선 하나 긋기
4. 전역 단축키로 그리기 ON/OFF (클릭 통과) 토글
5. 메뉴바 상주 + Dock 숨김 + Quit
6. 권한 처리 (Accessibility / Screen Recording)
7. 멀티 모니터
8. 서명 · 공증 · 배포
