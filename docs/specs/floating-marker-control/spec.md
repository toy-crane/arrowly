# 플로팅 마커 도구·속성 위계

## 목적

플로팅 마커의 기본 캡슐을 작게 유지하면서 펜·텍스트가 같은 계열의 도구이고, 색·굵기·텍스트 크기는 각 도구의 속성이라는 관계를 한눈에 이해하게 한다.

승인된 화면은 [prototype.html](./prototype.html)이다. 구현 시 시각·상태 전이의 기준으로 사용하되 프로덕션 코드를 복사하지 않는다.

## 확정 결정

- 기본 캡슐의 1층에는 `자유곡선 도구`, `텍스트 도구`, `블랙보드` 세 버튼만 둔다.
- 자유곡선과 텍스트는 같은 위계의 상호 배타적 도구다. 블랙보드는 배경 전환이므로 구분선 뒤에 격리한다.
- 펜 도구 아이콘은 도구 모양이 아닌 실제 출력인 자유곡선을 사용한다. 아이콘 선에는 현재 잉크색과 현재 굵기의 상대 단계를 함께 적용한다.
- 자유곡선 아이콘은 넓은 두 호가 한 번만 방향을 바꾸는 열린 스윕으로 고정한다. 작은 고리, 자기 교차, 추가 굴곡은 넣지 않는다.
- 활성 도구와 활성 블랙보드는 중립색 배경으로 표시한다. 색은 현재 잉크색을 나타내는 용도로만 사용한다.
- 2층은 현재 도구에 연결된 하나의 맥락형 속성 패널이다.
  - 자유곡선 도구: `색` 5개와 `굵기` 5단계
  - 텍스트 도구: `크기` `24/32/44/60/80px`를 나타내는 비례형 `T` 표본 5개
- 텍스트 크기는 화면 공유·강의 중 청중이 읽는 짧은 단어를 기준으로 `24/32/44/60/80px`를 사용한다. 첫 기본값은 `32px`다.
- 5단계는 한 줄 인스펙터와 키보드 증감에 맞는 선택 수를 유지하고, 각 단계는 작은 주석부터 강한 강조까지 약 1.3배씩 커지는 범위를 제공한다.
- 접힌 텍스트 도구의 `T`는 현재 텍스트 크기를 반영한다. 실제 `24/32/44/60/80px`를 각각 `15/17/19/22/24px`로 압축해 표시한다.
- 텍스트 크기의 숫자는 화면에 표시하지 않는다. 비례해서 커지는 `T` 표본으로 상대적인 결과를 비교하고 정확한 `px` 값은 접근성 이름과 툴팁에만 제공한다.
- 속성 패널에는 `색·굵기·크기` 시각적 라벨을 표시하지 않는다. 패널과 각 선택지의 접근성 이름에는 속성 의미와 값을 유지한다.
- 캡슐은 기존과 동일하게 높이 `44px`, 셀 `42×32px`, 다크 반투명 재질을 유지한다.
- 캡슐은 좌우 외곽 `8px`, 자유곡선–텍스트 사이 `4px`, 구분선 좌우 `8px`를 사용하며 전체 폭은 `163px`다.
- 속성 패널은 고정 최소 폭 없이 콘텐츠 크기를 따르며 외곽 패딩 `8px`, 선택지 사이 `4px`, 행 사이 `4px`를 사용한다.
- 기본 위치는 좌하단이다. 최상단 근처에서는 속성 패널을 캡슐 아래로 뒤집는다.
- 좌우 경계에서는 패널 전체를 화면의 `6px` 안전 여백 안으로 클램프한다. 연결 화살표는 패널과 함께 고정하지 않고 실제 활성 도구 버튼의 중심을 가리키도록 패널 내부에서 재배치한다.

## 상호작용

- 비활성 자유곡선·텍스트 버튼을 누르면 해당 도구로 전환하고 패널은 접힌 상태를 유지한다.
- 이미 활성화된 도구 버튼을 다시 누르면 그 도구의 속성 패널을 열거나 닫는다.
- 속성 값을 고르면 즉시 적용하고 패널을 닫는다.
- 바깥 누르기, 그리기 시작, 블랙보드 전환, 키보드 도구 전환은 열린 패널을 닫는다.
- 키보드 텍스트 키 `T`는 기존처럼 텍스트 모드를 토글하되 속성 패널은 열지 않는다.
- 텍스트 편집 중에도 텍스트 버튼을 눌러 크기 패널을 열 수 있다. 크기를 고르면 편집 세션은 유지하고 패널만 닫는다.
- 자유곡선 버튼을 누르면 진행 중인 텍스트 편집을 기존 확정 규칙에 따라 끝내고 펜 모드로 전환한다.
- 블랙보드는 현재 도구와 독립적으로 켜고 끄며, 도구 선택은 유지한다.
- 캡슐 드래그, 4px 탭·드래그 임계값, 위치 저장은 기존 동작을 유지한다.

## 상태

- `펜 기본`: 자유곡선 도구 활성, 패널 닫힘
- `펜 속성`: 자유곡선 도구 활성, 색·굵기 패널 열림
- `텍스트 기본`: 텍스트 도구 활성, 패널 닫힘
- `텍스트 속성`: 텍스트 도구 활성, 다음 텍스트의 기본 크기 패널 열림
- `편집 60`: 텍스트 편집 세션과 해당 텍스트의 `60px` 크기 패널 표시
- `편집 적용 후`: 크기를 적용해 패널이 닫혀도 텍스트 편집 세션이 유지되는 상태
- `보드`: 현재 도구를 유지한 채 블랙보드 활성
- `상단 배치`: 최상단에서 패널이 아래로 열리는 상태
- `우측 하단`: 우측 경계에서 패널과 연결 화살표가 각각 보정되는 상태
- `우측 상단`: 상단 방향 반전과 우측 경계 보정이 함께 적용되는 상태

## 구현 메모

- `Marker`는 활성 도구, 열린 속성 종류, 현재 값과 변경 콜백을 소유한다.
- 하나의 `ToolInspector`가 공통 표면, 패딩, 행 배치, 연결 화살표, 실제 크기 측정, 좌우 클램프와 상하 반전을 소유하고 도구별 선택지는 자식 콘텐츠로 받는다.
- `ToolInspector`는 현재 오버레이에서만 사용하므로 `overlay` 도메인 안에 둔다. 다른 창에서도 필요해질 때만 `shared`로 이동한다.
- 자유곡선 아이콘은 `currentColor` 기반 SVG로 만들고 현재 잉크색을 주입한다. 선택 배경색과 아이콘 색을 결합하지 않는다.
- 아이콘의 굵기 5단계는 실제 캔버스 픽셀을 그대로 축소하지 않고 `2/2.6/3.2/4.2/5.6px`로 시각 매핑한다. 최대 단계에서도 스윕의 떨어진 구간이 서로 붙지 않아야 한다.
- 텍스트 인스펙터의 5개 표본은 단계 차이를 충분히 비교할 수 있도록 `13/16/20/25/30px`로 표시한다.
- 접힌 `T`는 도구 위계를 유지하도록 더 좁은 `15/17/19/22/24px` compact scale을 사용한다. 실제 출력값과 표시값은 단계 키로 대응하며 수학식을 런타임에 계산하지 않는다.
- 패널 X 좌표는 `clamp(anchorCenter - panelWidth / 2, 6, viewportWidth - panelWidth - 6)`로 정하고, 화살표 X 좌표는 보정된 패널 좌표를 기준으로 활성 도구 중심을 다시 계산한다.
- 접근성 이름은 `자유곡선 도구`, `텍스트 도구`, `블랙보드 전환`, `색 …`, `굵기 …`, `텍스트 크기 …`를 제공한다.

## 비범위

- 지우개·도형·선택 도구 추가
- 현재 굵기를 숫자·보조 배지·두 번째 아이콘으로 중복 노출
- 속성 패널 중첩 또는 3단 메뉴
- 캡슐 크기·드래그·저장 방식 변경

## 남은 위험

- 승인된 스윕의 5색 × 5굵기 조합은 실제 `44px` 네이티브 캡슐에서 시각 회귀 확인이 필요하다.
- non-activating `NSPanel`에서는 합성 포인터 이벤트가 실제 클릭을 대체하지 못할 수 있으므로 구현 후 네이티브 수동 검증이 필요하다.

## 인수 조건

- 접힌 캡슐에서 자유곡선·텍스트가 같은 위계의 도구로 보이고 블랙보드는 구분선 뒤에 보인다.
- 자유곡선과 텍스트의 활성 영역 사이에는 `4px` 간격이 있고 캡슐 전체 폭은 `163px`다.
- 자유곡선 아이콘만 보고 현재 잉크색과 상대 굵기를 알 수 있다.
- 자유곡선 아이콘은 모든 굵기에서 열린 스윕의 실루엣을 유지하며 획 구간끼리 붙지 않는다.
- 펜 속성 패널에는 색·굵기만, 텍스트 속성 패널에는 텍스트 크기만 나타난다.
- 속성 패널에는 시각적 라벨이 없지만 패널·행·선택지의 접근성 이름으로 의미를 구분할 수 있다.
- 텍스트 크기 패널에는 숫자 대신 비례해서 커지는 `T` 표본 5개가 나타나며 접근성 이름에는 정확한 `px` 값이 남는다.
- 접힌 `T`는 현재 실제 크기 `24/32/44/60/80px`에 따라 `15/17/19/22/24px`로 변하고 캡슐 크기는 유지된다.
- 활성 도구 재클릭, 값 선택, 바깥 누르기, 그리기 시작의 열림·닫힘 규칙이 일관된다.
- 텍스트 편집 중 크기를 바꿔도 편집 세션이 유지된다.
- 최상단·좌우 가장자리에서도 패널이 화면 밖으로 나가지 않고 연결 화살표가 활성 도구를 가리킨다.
