# 플로팅 마커 도구·속성 위계

## 목적

플로팅 마커를 작게 유지하면서 현재 그리기 도구와 잉크 속성을 한눈에 확인하고, 자유곡선·기하·텍스트·삭제와 블랙보드의 관계를 이해하게 한다.

## 1층 도구 위계

- 기본 캡슐의 1층에는 `현재 그리기 도구`, `텍스트 도구`, `마크 삭제 도구`, 구분선, `블랙보드`를 순서대로 둔다.
- 현재 그리기·텍스트·마크 삭제는 상호 배타적인 잠금 도구다. `⌘` 이동 탐색과 `⌥` 삭제 탐색은 현재 도구를 바꾸지 않는 순간 상호작용이다.
- 블랙보드는 배경 전환이므로 구분선 뒤에 격리하고 현재 도구와 독립적으로 켜고 끈다.
- 현재 그리기 도구 버튼은 자유곡선·화살표·사각형·타원·삼각형 중 선택된 도구의 아이콘을 표시한다.
- 현재 그리기 도구 버튼의 아이콘은 현재 잉크색과 굵기의 상대 단계를 함께 반영한다.
- 활성 도구와 활성 블랙보드는 중립색 배경으로 표시한다. 잉크색은 현재 잉크 속성을 나타내는 용도로만 사용한다.
- 캡슐은 높이 `44px`, 셀 `42×32px`, 다크 반투명 재질과 기존 간격·위치 저장 규칙을 유지한다.

## 2층 도구 속성

- 2층은 현재 도구에 연결된 하나의 맥락형 도구 속성 패널이다.
- 현재 그리기 도구 패널은 세 줄이다.
  - 그리기 선택: `자유선`, `화살표`, `사각형`, `타원`, `삼각형`
  - 색: 옐로우·오렌지·핑크·그린·블루 5개
  - 굵기: 5단계
- 그리기 선택 아이콘은 도구 종류를 비교하기 위한 중립 흰색·고정 굵기로 표시한다.
- 텍스트 도구 패널은 `24/32/44/60/80px`를 나타내는 비례형 `T` 표본 5개다. 첫 기본값은 `32px`다.
- 텍스트 크기의 숫자는 화면에 표시하지 않고 정확한 `px` 값은 접근성 이름과 툴팁에만 제공한다.
- 패널에는 시각적 행 라벨을 표시하지 않으며 패널과 선택지의 접근성 이름으로 의미와 값을 전달한다.
- 패널은 콘텐츠 크기를 따르고 화면의 `6px` 안전 여백 안에서 위·아래 배치와 좌우 클램프를 적용한다.
- 마크 삭제 도구에는 별도 속성 패널을 두지 않는다.

## 상호작용

- 현재 그리기 도구 버튼을 누르면 선택 도구를 바꾸지 않고 그리기 속성 패널을 열거나 닫는다.
- 화살표·사각형·타원·삼각형을 고르면 해당 도구가 다른 도구를 고를 때까지 유지된다.
- 마크 하나를 확정해도 자유곡선으로 자동 복귀하지 않는다.
- 자유곡선을 고를 때만 자유곡선 도구로 명시적으로 복귀한다.
- 색·굵기·텍스트 크기를 고르면 즉시 적용하고 패널을 닫는다.
- 비활성 텍스트 버튼을 누르면 텍스트 도구로 전환하고, 이미 활성인 텍스트 버튼을 누르면 텍스트 크기 패널을 열거나 닫는다.
- 마크 삭제 버튼은 [마크 도구 선택·이동·삭제 일관성](../mark-tool-consistency/spec.md)의 잠금형 삭제 탐색으로 진입한다.
- 바깥 누르기, 그리기 시작, 블랙보드 전환과 키보드 도구 전환은 열린 패널을 닫는다.
- 텍스트 편집 중 텍스트 크기 선택은 세션을 유지한다. 다른 잠금 도구를 고르면 기존 텍스트 확정·취소 계약을 따른다.
- 블랙보드를 켜고 꺼도 현재 도구는 유지한다.
- 캡슐 드래그, `4px` 탭·드래그 임계값과 위치 저장은 기존 동작을 유지한다.

## 인수 조건

- 접힌 캡슐에서 현재 그리기·텍스트·마크 삭제가 같은 위계로 보이고 블랙보드는 구분선 뒤에 보인다.
- 그리기 패널에서 여섯 도구를 확정 순서로 고를 수 있다.
- 접힌 현재 도구 아이콘은 도구·색·굵기를 반영하고 패널의 도구 아이콘은 고정 굵기를 유지한다.
- 같은 기하 도구로 두 마크를 연속 생성해도 현재 도구 아이콘과 선택 상태가 유지된다.
- 현재 도구 버튼을 다시 눌러도 자유곡선으로 돌아가지 않고 패널만 열고 닫힌다.
- 텍스트 패널은 다섯 크기의 상대 차이를 보여주고 마크 삭제에는 속성 패널이 없다.
- 마커를 화면 모서리로 옮겨도 패널이 잘리거나 연결 화살표가 잘못된 도구를 가리키지 않는다.

## 비범위

- 선택 핸들, 도형 편집 패널과 부분 지우개
- 도구별 독립 색·굵기 저장
- 기하 도구의 회전·방향 제한 옵션

## 남은 위험

- 여섯 개의 그리기 선택지가 한 줄에 놓일 때 작은 화면에서 패널 폭과 안전 여백이 충분한지 실제 오버레이로 확인한다.
- 굵은 도형 아이콘이 `24×24px` 안에서 뭉개지지 않도록 다섯 굵기의 상대 표현을 시각 검증한다.
