> The code is the terrain and this plan is a map: where they disagree, the
> terrain wins. A divergence at the decision level flows back to spec.md
> instead of being worked around.

## Approach

공유 drawing 도메인에 모든 마크의 hit testing과 평행이동을 모으고, 기존 base/live 캔버스 구조로 B focus field를 구현한다.

- 직선과 닫힌 도형을 의미상 다른 타입으로 정리하되 런타임 표현은 불필요하게 바꾸지 않는다.
- 최상단 마크 찾기, 마크 평행이동, 후보 강조 경계를 순수 drawing 함수로 제공한다.
- 펜·직선은 실제 경로 주변의 넓어진 hit 영역을 사용하고 사각 경계 내부의 빈 공간은 잡지 않는다.
- 텍스트·도형은 기존 레이아웃과 기하 경계를 사용한다.
- 이동 이력은 저장소의 동일 인덱스 교체를 재사용해 z-order를 보존한다.
- `⌘` 상태는 대기·탐색·후보·이동 중으로 명시해 단축키와 포인터 입력 우선순위를 고정한다.
- base에는 일반 마크를, live에는 scrim·후보 강조·이동 미리보기를 그린다.

DOM 선택 요소, 세 번째 캔버스, 올가미 선택은 현재 범위에 비해 상태와 포인터 충돌을 지나치게 늘려 사용하지 않는다.

## Order

첫 번째 착륙점은 drawing 도메인이다. 모든 마크의 hit testing·평행이동·최상단 판정·이력 보존을 테스트로 고정한다. 여기까지는 화면 동작을 바꾸지 않는다.

두 번째 착륙점은 오버레이다. `⌘` `120ms` 대기와 단축키 억제, 공통 이동 제스처, B 렌더링을 연결한다. 자동 테스트와 밀집 화면 성능 측정 뒤 실제 macOS 오버레이에서 최종 검증한다.

## Acceptance criteria

- [ ] 모든 확정 마크가 `⌘` 드래그로 이동한다.
- [ ] 진행 중 획과 보정 미리보기는 이동 후보가 아니다.
- [ ] `⌘` 단독 `120ms` 뒤 focus field가 표시된다.
- [ ] `⌘Z`, `⌘+`, `⌘-`에서는 focus field가 나타나지 않는다.
- [ ] `120ms` 전에도 `⌘` 드래그는 즉시 후보가 된다.
- [ ] 펜·직선은 경로 강조, 텍스트·도형은 실선 프레임으로 표시된다.
- [ ] 가는 획은 잡기 쉽지만 획의 빈 사각 경계에서는 선택되지 않는다.
- [ ] `4px` 이하는 무변경이며 초과한 경우에만 이동한다.
- [ ] 빈 곳에서 `⌘` 드래그해도 새 마크가 생기지 않는다.
- [ ] 중간 `⌘` 해제와 pointer cancel이 계약대로 수렴한다.
- [ ] 확정·취소·Undo/Redo가 위치와 z-order를 보존한다.
- [ ] 텍스트 편집, 펜 입력, 직선 잠금, 전체 지우기와 Esc가 회귀하지 않는다.

## Verification and seams

- drawing 공개 API 단위 테스트: 네 종류 마크의 hit, 평행이동, 겹침 우선순위
- 저장소 공개 API 테스트: 무변경, 교체, Undo/Redo와 z-order
- 오버레이 상호작용 테스트: fake timer 기반 `120ms` 상태와 단축키 조합
- 포인터 상호작용 테스트: 도구 독립성, `4px` 임계값, 중간 keyup, 취소
- 캔버스 렌더 테스트: B 합성 순서와 alpha 상태 복원
- 밀집 장면 측정: `500`개 마크·`50,000`개 포인트에서 표시와 hover가 한 프레임을 지속적으로 넘기지 않는지 확인
- 전체 게이트: `bun run test:all`
- 네이티브 검증: `bun run tauri dev`에서 밝은 화면·어두운 화면·블랙보드와 실제 마우스 확인

## Off-limits

- 여러 마크 선택, 올가미, 자동 그룹
- 이동 뒤 지속되는 선택과 핸들
- 펜 경로·직선 끝점·도형 기하 편집
- 정렬, 스냅, 화면 경계 제한
- 단축키 설정이나 이동 전용 도구
- Rust IPC와 창 활성화 정책

## Risks and open questions

- 스무딩된 펜 경로와 원본 포인트 기반 hit 영역이 어긋날 수 있어 실제 곡선에서 검증해야 한다.
- 모든 후보 재렌더링이 느리면 focus 상태용 캐시 합성을 도입한다.
- `Meta` keyup 유실은 blur, 모드 전환, 취소와 unmount에서 강제 초기화한다.
- 이동 후에도 z-order를 보존하므로 다른 마크 뒤로 들어간 대상이 사라진 것처럼 보일 수 있다.
- 현재 막힌 제품 결정은 없다.
