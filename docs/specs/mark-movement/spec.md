# 마크 이동

## 목적

강의 흐름을 멈추는 지속적인 선택 모드 없이, 잘못 배치한 마크 하나의 위치를 즉시 교정한다. 평소 펜 입력은 그대로 유지하고 사용자가 `⌘`으로 이동 의도를 명시한 짧은 순간에만 마크를 이동 후보로 드러낸다.

## 이동 경계

- 펜 마크, 직선 마크, 도형 마크와 텍스트 마크를 포함한 모든 확정 마크는 이동할 수 있다.
- 진행 중 획과 보정 미리보기는 아직 마크가 아니므로 이동할 수 없다.
- 향후 추가되는 마크도 별도 예외 없이 이동할 수 있다.
- 펜 마크 하나는 한 번의 포인터 누름부터 놓기까지 확정된 획 하나다. 여러 펜 마크를 묶어 이동하지 않는다.
- 겹친 마크를 가리키면 화면에서 가장 위에 그려진 마크 하나를 대상으로 한다.
- 이동은 한 번에 마크 하나만 대상으로 하며 지속적인 선택 상태를 만들지 않는다.

## 이동 탐색 상태

- 그리기 모드에서 텍스트 편집 세션과 활성 포인터 제스처가 없을 때 `⌘`을 다른 키 없이 `120ms` 유지하면 이동 탐색 상태가 시작된다.
- `⌘`을 누른 뒤 다른 비수식키 입력이 이어지면 `⌘`을 놓을 때까지 이동 탐색 상태를 표시하지 않는다. `⌘Z`, `⌘+`, `⌘-` 같은 단축키는 focus field를 드러내지 않는다.
- 이동 탐색 상태에서는 화면의 대비를 일시적으로 낮추고 모든 마크를 전면에 강조한다.
- 펜 마크와 직선 마크는 실제 경로를 따라 애니메이션 없는 강조를 표시한다. 포인터 hit 영역은 가는 획도 잡을 수 있도록 보이는 경로보다 넓지만 마크의 사각 경계 전체로 확장하지 않는다.
- 텍스트 마크와 도형 마크는 실제 hit 영역을 포함하는 애니메이션 없는 실선 프레임으로 강조한다.
- 마크의 원래 색과 형태는 식별할 수 있어야 하며, 포인터를 올린 마크의 강조를 더 강하게 표시하고 이동 커서를 보여준다.
- `⌘`을 놓으면 이동 탐색 상태와 후보 강조를 즉시 제거한다. 선택 상태나 핸들을 남기지 않는다.
- 이동이 시작된 뒤 `⌘`을 놓으면 후보 전체의 강조는 제거하되 이동 중인 마크의 경로 강조나 프레임은 포인터를 놓거나 입력이 취소될 때까지 유지한다.
- 후보와 hover 강조에는 반복 애니메이션을 사용하지 않는다.

## 마크 이동 제스처

- 포인터를 누르는 순간 `⌘`이 눌려 있고 마크를 hit한 경우에만 마크 이동 제스처의 후보가 된다. 현재 선택된 도구와 `120ms` 표시 지연은 진입 조건에 영향을 주지 않으며, 대상 강조는 즉시 표시한다.
- 포인터를 누른 뒤 `⌘`을 추가로 눌러도 이동으로 전환하지 않는다.
- `⌘`을 누른 채 빈 곳을 누르면 펜 마크나 텍스트 초안을 만들지 않고 아무 동작도 하지 않는다.
- 후보가 된 뒤 포인터 이동이 `4px` 이내면 놓을 때 아무 동작도 하지 않고, `4px`을 넘으면 마크 이동 제스처로 전환한다.
- 이동은 잡은 지점과 마크 기준점의 상대 위치를 유지하고 새 위치를 실시간으로 보여준다. 원래 위치의 마크를 중복 표시하지 않는다.
- 이동이 시작된 뒤에는 `⌘`에서 손을 떼더라도 포인터를 놓거나 입력이 취소될 때까지 같은 제스처를 계속한다.
- 포인터를 놓으면 위치 변경을 실행 취소 한 단위로 기록하고 이동 전의 도구를 유지한다.
- 포인터 입력이 취소되면 원래 위치를 복원하고 이동 전의 도구를 유지한다.
- 화면 경계 제한이나 스냅은 적용하지 않는다.

## 이력

- 마크 하나의 이동은 실행 취소 한 단위다.
- 위치가 바뀌지 않은 제스처는 이력을 만들지 않는다.
- Undo/Redo는 이동 전후 위치와 원래 z-order를 보존한다.

## 자동 인수 조건

- `⌘`을 다른 키 없이 `120ms` 유지하면 모든 확정 마크가 전면에 강조된다.
- `⌘` 단축키 조합은 focus field를 표시하지 않고, `120ms` 전에 시작한 `⌘` 드래그는 지연 없이 이동 후보가 된다.
- 펜 마크와 직선 마크는 경로를 따라 강조되고 텍스트 마크와 도형 마크는 실선 프레임으로 강조된다.
- `⌘` 없이 시작한 입력은 마크 이동으로 전환되지 않는다.
- `⌘`을 누른 채 빈 곳에서 시작한 입력은 마크나 초안을 만들지 않는다.
- 마크에서 시작한 `4px` 이하 입력은 아무 변경도 만들지 않고, `4px` 초과 끌기만 이동으로 전환한다.
- 이동 중 `⌘`을 놓아도 대상 마크의 강조와 제스처는 유지되고 다른 후보의 강조는 사라진다.
- 이동 미리보기에는 원래 위치의 마크가 중복 표시되지 않는다.
- 이동 확정, 취소, Undo/Redo가 위치와 z-order를 복원한다.
- 겹친 마크에서는 가장 위에 그려진 마크 하나만 이동한다.
- 펜 입력, 텍스트 작성·교정, 전체 지우기, 직선 잠금과 두 단계 Esc가 회귀하지 않는다.

## 네이티브 인수 조건

다음은 실제 macOS 오버레이에서 확인한다.

1. `⌘`을 단독으로 누르면 `120ms` 뒤 focus field가 나타나고, `⌘`을 놓으면 즉시 사라지며 아래 앱이나 입력 포커스를 활성화하지 않는다.
2. `⌘Z`, `⌘+`, `⌘-`를 사용할 때 focus field가 번쩍이지 않고 각 단축키의 기존 동작이 유지된다.
3. 밝은 화면, 어두운 화면과 블랙보드에서 모든 마크의 후보 강조와 원래 색·형태를 식별할 수 있다.
4. 실제 마우스에서 `⌘` 없이 마크 위를 그으면 펜 입력이 유지되고, `⌘`을 누른 채 마크를 끌 때만 이동한다.
5. 펜 마크와 직선 마크는 빈 사각 경계가 아니라 실제 경로 근처에서 잡히며 가는 획도 무리 없이 hit할 수 있다.
6. 이동 중 `⌘`을 놓거나 포인터 입력을 취소했을 때 강조, 위치와 현재 도구가 계약대로 수렴한다.
7. 블랙보드에서도 모든 마크의 이동, 취소와 Undo/Redo가 동일하게 동작한다.

## 비범위

- 지속적인 선택 모드, 이동 전용 도구, 선택 핸들
- 여러 마크 동시 선택·이동, 그룹, 정렬과 스냅
- 여러 펜 마크를 하나의 글씨나 그림으로 묶는 자동 그룹과 올가미 선택
- 펜 마크의 경로 교정과 직선 마크의 끝점 교정
- 도형 마크의 크기 변경, 회전, 꼭짓점·끝점 교정
- 이동 탐색 상태를 유지하는 토글이나 사용자 설정
