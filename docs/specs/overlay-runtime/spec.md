# 오버레이 런타임

## 목적

화면 전체를 덮는 주석 오버레이가 아래 앱의 활성 상태를 해치지 않으면서 그리기와 통과를 안전하게 전환한다. 블랙보드는 별도 모드가 아니라 그리기 모드에 결합되는 배경 상태다.

## 그리기 모드와 통과 모드

- 그리기 모드에서는 주석 오버레이가 보이고 포인터와 키 입력을 받아 판서한다.
- 통과 모드에서는 주석 오버레이를 숨기고 포인터 입력을 아래 앱으로 전달한다.
- 통과 모드 전환은 숨김이며 삭제가 아니다. 다시 그리기 모드에 들어오면 기존 마크가 복원된다.
- 삭제는 실행 취소와 전체 지우기 같은 명시적 동작으로만 일어난다.
- 편집 세션이 없을 때 Esc를 누르면 즉시 통과 모드로 전환한다.

## 블랙보드

- 블랙보드는 그리기 모드의 배경을 불투명 검정으로 바꾸는 상태다.
- 블랙보드를 켜고 꺼도 같은 마크 집합을 사용한다. 화면 위 주석을 검은 판으로 가져가 이어서 설명할 수 있다.
- 통과 모드에서 블랙보드를 요청하면 그리기 모드와 블랙보드를 함께 켠다.
- 블랙보드 상태는 그리기 모드와 통과 모드를 오가며 유지하지만 앱을 재시작하면 투명 배경으로 초기화한다.
- 화이트보드, 반투명 블랙보드, 여러 블랙보드 페이지는 제공하지 않는다.
- 창 레벨을 메뉴바보다 높이지 않는다. 메뉴바를 통한 탈출 가능성이 배경의 완전한 화면 점유보다 우선한다.

## 화면 범위

- 그리기 모드에 들어가는 순간 커서가 있는 모니터 하나를 덮는다.
- 다른 모니터로 옮기려면 통과 모드로 나갔다가 그 모니터에서 다시 그리기 모드에 들어간다.
- 직전과 다른 모니터에서 그리기 모드가 켜지면 기존 좌표가 무효이므로 마크를 모두 지운다.
- 해상도 변경, 모니터 연결·해제, 잠자기 복귀, 클램셸 전환 뒤 오버레이를 현재 화면에 다시 맞춘다.

## 탈출 안전장치

- 오버레이가 키 입력을 받는 동안에도 아래 앱은 활성 상태로 보여야 한다.
- Esc 경로는 웹뷰와 독립된 전역 경로를 사용한다.
- 텍스트 편집 중 첫 Esc는 편집을 끝내고, 그다음 Esc는 통과 모드로 나간다.
- 웹뷰가 응답하지 않아 첫 Esc의 편집 종료 이벤트를 처리하지 못해도 다음 Esc로 탈출할 수 있어야 한다.
- 메뉴바 아이콘은 오버레이가 떠 있을 때도 클릭할 수 있어야 한다.
- 그리기 토글 단축키 등록 실패 등 탈출 수단을 확보하지 못한 상태에서는 그리기 모드에 들어가지 않고 이유를 알린다.

## 네이티브 인수 조건

다음 항목은 브라우저 모의 실행이나 코드 커버리지로 대체하지 않고 실제 macOS 앱에서 확인한다.

1. 웹뷰 배경까지 투명하고 그린 요소만 아래 화면 위에 보인다.
2. 그리기 모드에서 ⌘Z가 Arrowly에 도달하면서 아래 앱의 메뉴바와 창은 활성 상태로 보인다.
3. 전체화면 Space 위에 오버레이가 표시된다.
4. 오버레이가 떠 있어도 메뉴바를 클릭할 수 있다.
5. 통과 모드의 클릭이 아래 앱에 전달된다.
6. 웹뷰 응답을 멈춘 뒤에도 Esc로 통과 모드에 나갈 수 있다.
7. 실제 전역 단축키 충돌을 알리고 유효한 설정으로 복구할 수 있다.
8. 모니터 구성 변경과 잠자기 복귀 뒤 오버레이 위치가 맞고 좌표가 무효인 마크가 남지 않는다.

## 비범위

- 통과 모드에서 마크 표시 유지
- 여러 모니터 동시 오버레이
- 메뉴바와 Dock까지 덮는 창 레벨
- 접근성 이벤트 탭을 이용한 키 가로채기
