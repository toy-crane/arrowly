# 앱 셸과 학습 흐름

## 목적

평소에는 보이지 않는 메뉴바 유틸로 머물면서, 사용자가 언제든 안전하게 판서를 시작하고 설정을 바꾸며 종료할 수 있게 한다. 첫 실행에서는 짧은 체험으로 모드와 탈출 방법을 익힌다.

## 앱 형태

- 평소에는 Dock과 앱 스위처에서 숨고 메뉴바에만 상주한다.
- 온보딩·설정 같은 일반 창이 열려 있는 동안만 Dock에 표시하고, 마지막 일반 창이 닫히면 다시 메뉴바 전용 상태로 돌아간다.
- 메뉴바 메뉴에는 그리기, 블랙보드, 텍스트 입력, 전체 지우기, 마커 숨기기, 단축키 설정, 로그인 시 실행, 튜토리얼 다시 보기, 종료를 제공한다.
- 메뉴바 메뉴의 각 동작에는 현재 단축키를 표시해 상시 치트시트로 사용한다.
- 전체화면 앱에서는 메뉴바가 숨으므로 전역 단축키가 유일한 진입 경로가 될 수 있다.
- 메뉴바 아이콘은 다크·라이트 모드에 대응하는 템플릿 이미지다.
- 앱 번들 아이콘은 Finder, DMG, 시스템 설정, 로그인 항목에 노출된다. 원본은 [assets/icon.svg](../../../assets/icon.svg)다.

## 단축키 정책

- 그리기 토글 `⌥Tab`과 블랙보드 토글 `⇧⌥Tab`만 상시 전역 등록한다.
- 전체 지우기 `⌥⌫`와 텍스트 `T`는 그리기 모드에서 오버레이가 처리하고 전역 등록하지 않는다.
- ⌘Z/⇧⌘Z는 OS 표준 실행 취소 키로 유지하고 전역 등록하지 않는다.
- Esc, 마크 이동 `⌘` 드래그, 순간 삭제 `⌥` 클릭, 잠금형 마크 삭제 도구 `E`, 현재 도구 크기 `⌘+`/`⌘−`는 고정 키이며 재설정하지 않는다.
- 그리기, 블랙보드, 전체 지우기, 텍스트 네 단축키는 사용자가 재설정할 수 있다.
- 전역 단축키는 수식키가 필요하다. 로컬 텍스트 키는 수식키 없는 단독 키를 허용한다.
- Esc, ⌘Z 계열, 고정 이동·삭제 수식키 `⌘`·`⌥`, 고정 마크 삭제 키 `E`, Arrowly 내부 중복 조합은 허용하지 않는다. 기존 텍스트 단축키가 무수식 `E`이면 기본값 `T`로 마이그레이션한다.
- 전역 단축키 등록 실패는 조용히 넘기지 않고 해당 입력 옆에 이유를 표시한다.
- 다른 앱 내부 단축키와의 충돌은 감지할 수 없으므로 기본값과 사용자 재설정으로 완화한다.

## 온보딩

- 첫 실행에는 약 20초의 세 단계 인터랙티브 튜토리얼을 연다. 전체 기능을 소개하지 않고, 첫 판서를 안전하게 시작하고 교정한 뒤 끝내는 핵심 흐름만 실습한다.
- 1단계는 현재 그리기 시작 단축키를 참고용으로 보여주되 입력을 요구하지 않는다. 사용자는 미니 캔버스에 마크 하나를 그려야 다음 단계로 이동한다.
- 2단계는 같은 마크를 `⌘` 드래그로 옮기고, `⌥` 클릭으로 지운 뒤, `⌘Z`로 되돌리는 순서로 진행한다. 각 동작이 완료되면 다음 동작을 안내한다.
- 3단계는 현재 전체 지우기 단축키로 미니 캔버스를 비운 뒤 `Esc`를 안내한다. `Esc`를 누르면 온보딩 완료를 저장하고 별도 완료 화면 없이 창을 닫는다.
- 블랙보드, 텍스트, 포인터 핑, 잠금형 마크 삭제 도구 `E`, 현재 도구 크기 변경은 필수 온보딩에서 실습하지 않는다.
- 온보딩에서는 단축키를 변경하지 않는다. 변경은 설정 창만 담당한다.
- 온보딩이 끝나거나 창이 닫히면 임시로 해제한 전역 단축키를 복구한다.
- 메뉴바에서 언제든 튜토리얼을 다시 시작할 수 있다.

## 단축키와 제스처 설정

- 설정 창은 한 화면의 `바꿀 수 있는 단축키`와 `고정 조작` 두 세로 섹션으로 구성한다. 탭이나 접힌 영역으로 나누지 않으며, 작은 창에서는 세로 스크롤을 허용한다.
- `바꿀 수 있는 단축키`는 `그리기 시작·끝내기`, `블랙보드 켜기·끄기`, `마크 모두 지우기`, `텍스트 쓰기` 네 행이다.
- 각 단축키 필드는 현재 조합을 표시한다. 필드를 누르면 새 조합을 입력하며, 행별 복구 동작은 `기본값`으로 표기한다.
- `고정 조작`은 `마크 옮기기`, `마크 하나 지우기`, `되돌리기`, `다시 실행`, `마크 연속 지우기`, `도구 크기 조절`, `그리기 끝내기`를 표시한다.
- `⌥` 클릭의 `마크 하나 지우기`와 `E`의 `마크 연속 지우기`를 서로 다른 행으로 보여 일회성 삭제와 잠금형 마크 삭제 도구를 구분한다.
- 고정 조작은 참고용이며 필드·`기본값` 동작·`고정` 배지를 표시하지 않는다.
- 화면 제목은 `단축키와 제스처`다. 사용자용 레이블은 짧은 행동형 문구를 사용하고, 내부 상태명이나 `전환` 같은 구현 중심 표현을 노출하지 않는다.
- 단축키 입력 오류는 해당 행 바로 아래에 원인과 해결 방법을 표시한다.

확정된 화면 구조와 주요 상태는 [프로토타입](./prototype.html)을 따른다.

## 설정 영속화

- 잉크 색, 펜 굵기, 다음 텍스트 기본 크기, 플로팅 마커 위치와 숨김 상태를 저장한다.
- 네 개의 사용자 단축키와 온보딩 완료 여부를 저장한다.
- 블랙보드 상태와 현재 마크는 앱을 재시작할 때 복원하지 않는다.
- 이전 버전의 설정에 새 필드가 없으면 안전한 기본값으로 보완한다.

## 시각 언어

- 일반 창은 웜 페이퍼 배경과 잉크색 텍스트, 형광 옐로우 포인트를 사용하는 “잉크 & 종이” 테마다.
- 다크 모드에서는 잉크 배경과 형광 포인트의 관계를 반전한다.
- 일반 창의 UI 크롬에는 판서용 다섯 잉크색을 장식으로 사용하지 않는다.
- 화면 위의 플로팅 마커와 커서는 시스템 테마와 무관하게 일관된 대비를 유지한다.

## 인수 조건

- 앱을 실행하면 일반 창 없이 메뉴바에 상주한다.
- 온보딩 또는 설정 창이 열릴 때 Dock과 표준 앱 메뉴가 나타나고 마지막 창이 닫히면 사라진다.
- 메뉴 항목의 단축키 표시가 저장된 설정과 일치한다.
- 단축키 입력기는 예약 키, 내부 중복, 전역 등록 실패를 구분해 표시한다.
- 설정 창에서 변경 가능한 단축키와 고정 조작이 서로 다른 섹션으로 구분되고, 일회성 삭제와 잠금형 삭제의 차이가 레이블로 드러난다.
- 온보딩에서 그리기, 이동, 삭제, 되돌리기, 전체 지우기와 `Esc` 완료를 순서대로 체험할 수 있다.
- 온보딩의 `Esc` 완료 뒤 창이 닫히고 온보딩 완료 상태가 저장된다.
- 설정 저장 뒤 재시작해도 영속 대상 값이 복원되고 블랙보드와 마크는 초기화된다.
- 메뉴바에서 그리기 시작, 전체 지우기, 텍스트 입력, 종료가 모두 가능하다.

## 비범위

- Dock에 상시 표시되는 일반 앱 형태
- 일반 창이 열리지 않은 상태의 표준 ⌘Q 의존
- 자동 업데이트
