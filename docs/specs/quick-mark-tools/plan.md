# 빠른 마크 도구 구현 계획

## Approach

현재의 `textMode` 불리언을 자유곡선·텍스트·마크 삭제·일회성 빠른 추가를 표현하는 명시적인 도구 상태로 교체한다.

- `OverlayApp`이 현재 도구와 잉크 속성의 단일 소스를 소유한다.
- 마크 모델에 삼각형과 직선의 끝점 화살촉 옵션을 추가한다.
- `DrawingCanvas`가 빠른 추가, 삭제, 더블클릭, 홀드 보정의 입력 우선순위와 이력을 관리한다.
- `Marker`는 `209px` 도구 위계와 3줄 자유곡선 패널을 표현한다.
- 포인터 핑은 마크 저장소와 분리된 `PointerPingLayer`에서 독립 인스턴스로 재생한다.
- 고정 로컬 키 `E`는 웹뷰에서 처리하며 네이티브 단축키·설정 스키마는 확장하지 않는다.

배제한 접근:

- 기존 `textMode`에 여러 불리언 추가: 충돌 가능한 상태 조합이 늘어난다.
- 화살표를 별도 도형으로 모델링: 직선 끝점 옵션이라는 확정안과 어긋난다.
- 하나의 핑 노드를 재시작: 빠른 연속 입력의 중첩이 불가능하다.
- 직선·도형 이동 유지: 삭제와 Undo만 허용한 최신 제품 경계와 충돌한다.

## Order

1. 첫 구현 세션은 도메인 모델과 순수 계산 경계를 완성한다. 도구 상태, 삼각형 기하, 끝점 화살촉, 렌더링·hit testing·이력, 이동 가능 마크 제한을 추가한다. 이 단계에서 마크 모델 단위 테스트가 통과해야 한다.
2. 다음 세션은 플로팅 마커와 포인터 제스처를 연결한다. 빠른 추가의 한 번 사용 후 복귀, `4px` 취소 임계값, `Shift` 각도 제한, 지속 삭제, `⌘+`/`⌘−`, `E` 입력을 구현한다. 컴포넌트 및 캔버스 상호작용 테스트로 검증한다.
3. 마지막 세션은 홀드 보정과 더블클릭 흐름을 완성한다. `350ms` 보정, 삼각형 분류, 움직임 취소, 텍스트 편집 우선순위, 독립 포인터 핑, 온보딩·설정 안내 문구를 연결한다. 전체 자동 게이트 후 실제 macOS 오버레이에서 수동 인수 검증한다.

## Acceptance criteria

- [ ] 플로팅 마커가 `자유곡선 · 텍스트 · 마크 삭제 | 블랙보드` 순서와 `209px` 폭을 따른다.
- [ ] 자유곡선 패널에서 다섯 빠른 추가를 선택할 수 있다.
- [ ] 빠른 추가는 성공한 마크 하나 뒤 자유곡선으로 복귀한다.
- [ ] 직선·화살표는 `Shift`로 45° 제한할 수 있다.
- [ ] 홀드 보정은 `350ms`에 사각형·타원·삼각형·직선만 미리보기한다.
- [ ] 보정 미리보기 뒤 움직이면 자유곡선으로 돌아가고 화살표는 자동 생성하지 않는다.
- [ ] `T` 뒤 클릭은 새 텍스트, 텍스트 더블클릭은 교정, 그 밖의 더블클릭은 포인터 핑이 된다.
- [ ] 빠른 핑 입력 세 번이 취소 없이 독립적으로 겹쳐 재생된다.
- [ ] `E` 또는 1층 버튼으로 진입한 삭제 도구가 여러 번 삭제해도 유지된다.
- [ ] 삭제 Undo/Redo 후에도 삭제 도구가 유지되고 원래 z-order가 복원된다.
- [ ] `⌘+`/`⌘−`가 자유곡선·빠른 추가의 굵기와 텍스트 크기를 올바르게 바꾼다.
- [ ] 직선·도형은 이동하지 않고 펜·텍스트만 기존 이동 제스처를 유지한다.
- [ ] 전체 지우기, 블랙보드, 텍스트 IME, 두 단계 Esc가 회귀하지 않는다.

## Verification and seams

- 마크 모델: 삼각형·화살촉의 렌더링, hit testing, 삭제 이력과 z-order 단위 테스트
- 보정 분류기: 실제 마우스에 가까운 사각형·타원·삼각형·직선 및 퇴화 표본 회귀 테스트
- `DrawingCanvas`: fake timer와 제한된 rAF 스텁으로 `350ms` 보정, 취소, 도구 우선순위, Undo/Redo 검증
- `Marker`: 역할·접근성 이름·활성 상태·패널 클램핑·`209px` 구조 검증
- 포인터 핑: Web Animations 완료 Promise를 제어해 `0 → 3 → 2 → 1 → 0` 독립 수명 검증
- `OverlayApp`: 도구 상태·커서·속성 저장의 통합 검증
- 자동 게이트: `bun run test:all`
- 네이티브 검증: 실제 `NSPanel`에서 빠른 더블클릭, IME 중 `E`, 연속 삭제, 핑 중첩, 도형 1회 복귀 확인

## Off-limits

- Rust IPC·이벤트와 단축키 저장 스키마 확장
- 지속 선택, 조절 핸들, 직선·도형 이동·리사이즈
- 시작점·양방향 화살촉과 화살표 자동 인식
- 부분 지우기와 드래그 연속 삭제
- 핑 색·크기·속도 설정
- 캔버스 렌더링 구조의 전면 교체
- 원격 push

## Risks and open questions

- 삼각형과 사각형의 손그림 모서리 수가 겹칠 수 있어 실제 포인터 표본이 필요하다.
- `350ms`가 실제 강의 중 멈춤을 오인하면 수치만 임의 조정하지 않고 스펙으로 되돌려 재결정한다.
- 더블클릭 첫 클릭의 점 마크가 회수되기 전 짧게 보일 수 있다.
- WKWebView의 빠른 연속 입력과 Web Animations 완료 순서는 네이티브 검증이 필요하다.
- 현재 추가 제품 결정은 없다.
