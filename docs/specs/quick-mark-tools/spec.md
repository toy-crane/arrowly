# 포인터 핑과 마크 삭제 보조 진입

## 목적

화면의 한 지점을 저장 없이 짧게 강조하고, 마크 삭제를 플로팅 마커와 키보드에서도 발견할 수 있게 한다.

그리기 도구의 선택·지속 상태, 이동·삭제 탐색, hit-test와 이력 계약은 [마크 도구 선택·이동·삭제 일관성](../mark-tool-consistency/spec.md)이 소유한다.

## 텍스트와 더블클릭

- 기본 텍스트 키 `T`로 텍스트 도구를 준비하고 원하는 위치를 한 번 클릭해 새 텍스트 편집 세션을 시작한다.
- 기존 텍스트 마크는 현재 도구와 관계없이 더블클릭하면 교정한다.
- 빈 곳 또는 텍스트가 아닌 마크를 더블클릭하면 해당 위치에 포인터 핑을 재생한다. 재생 시점은 두 번째 클릭을 누른 순간이며 떼기를 기다리지 않는다. 텍스트 마크 교정은 종전대로 두 번째 클릭을 뗄 때 시작한다.
- 이동·삭제 탐색 또는 잠긴 삭제 도구가 포인터 입력을 소유하는 동안에는 더블클릭 핑을 만들지 않는다.
- 더블클릭의 첫 클릭이 만든 점 펜 마크는 두 번째 클릭에서 회수해 마크와 실행 취소 이력을 남기지 않는다.

## 포인터 핑

- 포인터 핑은 옐로우 `#FFD400` 리플이다. 클릭한 지점에 작은 중심 점이 짧게 맺히고, 그 둘레로 링 두 겹이 바깥으로 퍼지며 사라진다.
- 중심 점은 약 `340ms` 만에 맺혔다 사라지고, 링은 약 반경 `16px`까지 확장한다. 안쪽 링은 조금 늦게 출발해 더 작은 반경에서 멈춘다. 전체 수명은 `560ms`다.
- 링은 두께 `2px` 테두리이며 안을 채우지 않는다.
- 링은 반경 0에서 출발해 바깥으로 자란다. 중심 점의 가장자리를 벗어나는 순간 최대 불투명도에 닿고, 그 전 구간은 중심 점에 가려 보이지 않는다. 이미 벌어진 반경에서 나타나지 않는다.
- 중심 점은 링보다 먼저 최대 밝기에 닿는다. 발원점이 켜진 뒤 파문이 나온 것으로 읽혀야 한다.
- 입력 한 번마다 독립된 리플 하나를 새로 만든다. 이전 리플을 취소하거나 다시 시작하지 않는다.
- 빠른 연속 입력은 동시에 겹쳐 재생한다. 각 리플은 이전과 반경을 소폭 달리해 같은 자리에 겹쳐도 구분된다.
- 포인터 핑은 hit testing을 가로막지 않으며 마크, 실행 취소 이력, 전체 지우기 대상과 저장 상태에 포함되지 않는다.
- 동작 줄이기 설정에서는 한 개의 정적 링이 `150ms` 안에 나타났다 사라지며 공간 확장을 사용하지 않는다.

## 마크 삭제 보조 진입

- `⌥`을 누른 동안의 순간 삭제 탐색이 기본 빠른 경로다.
- 플로팅 마커의 마크 삭제 버튼과 로컬 키 `E`는 같은 삭제 탐색 화면을 잠금 상태로 여는 보조 진입이다.
- 잠금 상태는 다른 도구를 고르거나 `E`를 다시 누를 때 끝난다.
- 텍스트 편집 중에는 글자 입력이 우선하고 `E`가 삭제 탐색을 시작하지 않는다.

## 인수 조건

- 텍스트 더블클릭은 교정을 시작하고 그 밖의 더블클릭은 포인터 핑을 만든다.
- 포인터 핑을 빠르게 세 번 입력하면 독립된 리플 세 개가 겹쳐 보이고 각각 자신의 `560ms` 수명이 끝날 때 제거된다.
- 두 번째 클릭을 길게 누르고 있어도 핑은 누른 즉시 재생되고, 떼는 시점에 다시 재생되지 않는다.
- 이동·삭제 제스처와 삭제 잠금 상태에서는 포인터 핑을 만들지 않는다.
- `E`와 마크 삭제 버튼이 같은 배경 딤·후보·hit-test를 사용하고 `E` 재입력으로 잠금을 끝낸다.

## 비범위

- 포인터 핑의 색·반경·속도 설정
- 지속적인 커서 하이라이트와 스포트라이트
- 빈 곳 더블클릭을 통한 새 텍스트 작성
