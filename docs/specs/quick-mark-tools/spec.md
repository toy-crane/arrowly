# 빠른 마크 도구와 포인터 핑

## 목적

강의 중 자유곡선 흐름을 유지하면서도 또렷한 도형·선·화살표를 한 번에 추가하고, 잘못 만든 마크를 연속해서 지우며, 화면의 한 지점을 저장 없이 짧게 강조한다.

승인된 화면과 동작은 [prototype.html](./prototype.html)이다. 구현 시 도구 위계·상태 전이·포인터 핑의 시각 기준으로 사용하되 프로덕션 코드를 복사하지 않는다.

## 도구 위계

- 플로팅 마커의 1층은 `자유곡선 도구`, `텍스트 도구`, `마크 삭제 도구`, 구분선, `블랙보드` 순서다.
- 자유곡선·텍스트·마크 삭제는 상호 배타적 도구다. 블랙보드는 현재 도구와 독립된 배경 상태다.
- 자유곡선 도구의 속성 패널 첫 줄은 사용 빈도에 따라 `자유선`, `화살표`, `사각형`, `타원`, `삼각형` 순서로 둔다. 자유선은 기본 도구 복귀이고 나머지 네 개만 빠른 추가다. 직선은 첫 줄에서 제외하고 홀드 보정으로만 만든다. 다음 줄은 색, 마지막 줄은 굵기다.
- 화살표 빠른 추가는 별도 도형 종류가 아니라 끝점 화살촉이 있는 직선 마크를 만든다. 시작점·양방향 화살촉은 제공하지 않는다.
- 빠른 추가를 고르면 자유곡선 버튼이 해당 아이콘과 `1회` 상태를 표시한다. 한 마크를 성공적으로 확정하면 자유곡선 도구로 자동 복귀한다.
- 플로팅 마커는 기존 높이 `44px`, 셀 `42×32px`, 외곽·도구 간격 규칙을 유지하며 1층 폭은 `209px`다.

## 빠른 추가 제스처

- 빠른 추가가 준비된 상태에서 빈 곳을 드래그하면 현재 잉크 색과 굵기로 미리보기를 그리고, 포인터를 놓을 때 마크 하나로 확정한다.
- 사각형과 타원은 시작점과 끝점의 경계 상자에 맞춘다. 삼각형은 같은 경계 상자 안의 위쪽 꼭짓점과 아래쪽 밑변을 가진 이등변 삼각형이다.
- 화살표는 시작점을 고정하고 현재 포인터를 끝점으로 사용하며 화살촉은 끝점에만 그린다.
- 화살표를 드래그하는 동안 Shift를 누르면 끝점 방향을 45° 단위로 제한한다. Shift를 놓으면 같은 포인터 위치를 향하는 자유 각도로 돌아간다.
- `4px` 이하의 입력은 마크를 만들지 않고 빠른 추가 준비 상태를 유지한다.
- 포인터 입력이 취소되면 미리보기를 버리고 빠른 추가 준비 상태를 유지한다.
- 다른 도구 선택, 전체 지우기, 그리기 모드 종료는 빠른 추가 준비와 진행 중 미리보기를 취소한다.
- 확정된 직선·도형 마크는 이동·끝점·크기·회전·꼭짓점을 편집하지 않고 삭제와 실행 취소로만 바로잡는다. 펜·텍스트 마크는 기존 위치 교정 계약을 유지하고 텍스트만 내용과 크기를 추가로 교정한다.

## 홀드 보정

- 자유곡선 도구에서 진행 중 획의 끝이 `3.5px` 반경 안에 `350ms` 머물면 보정 미리보기를 표시한다.
- 멈춘 지 `150ms`부터 진행 링을 표시하고 `350ms`에 사각형·타원·삼각형·직선 중 하나의 미리보기로 바꾼다.
- 미리보기가 보이는 동안 포인터를 놓으면 보정된 마크 하나로 확정한다.
- 미리보기가 보인 뒤 포인터가 `3.5px`를 벗어나면 보정을 취소하고 같은 진행 중 획의 자유곡선 표시로 돌아간다.
- 닫힌 획은 사각형·타원·삼각형만, 열린 획은 직선만 후보가 된다. 화살표는 자동으로 만들지 않고 빠른 추가로만 만든다.
- 짧거나 퇴화한 획, 판정 신뢰도가 낮은 획은 보정하지 않고 펜 마크로 확정한다.
- 홀드 전에 놓으면 어떤 보정도 하지 않는다.

세부 분류·취소 계약은 [홀드 보정 제스처](../line-lock-gesture/spec.md)를 따른다.

## 텍스트와 더블클릭

- 기본 텍스트 키 `T`로 텍스트 도구를 준비하고 원하는 위치를 한 번 클릭해 새 텍스트 편집 세션을 시작한다.
- 기존 텍스트 마크는 현재 도구와 관계없이 더블클릭하면 교정한다.
- 빈 곳 또는 텍스트가 아닌 마크를 더블클릭하면 해당 위치에 포인터 핑을 재생한다. 새 텍스트 편집 세션은 시작하지 않는다.
- 마크 삭제 도구와 `⌘` 마크 이동 제스처가 포인터 입력을 소유하는 동안에는 더블클릭 핑을 만들지 않는다.
- 더블클릭의 첫 클릭이 만든 점 펜 마크는 두 번째 클릭에서 회수해 마크와 실행 취소 이력을 남기지 않는다.

## 포인터 핑

- 포인터 핑은 옐로우 `#FFD400` 마름모 파티클 8개가 입력점을 중심으로 8방향으로 퍼지는 버스트다.
- 파티클은 약 `200ms` 안에 반경 `34px`에 도달하고, 그 위치에서 약 `300ms` 동안 잔상만 사라진다. 전체 수명은 `500ms`다.
- 입력 한 번마다 독립된 버스트 하나를 새로 만든다. 이전 버스트를 취소하거나 다시 시작하지 않는다.
- 빠른 연속 입력은 동시에 겹쳐 재생한다. 각 버스트는 이전 버스트와 경로가 구분되도록 방향을 조금씩 회전하고 반경을 소폭 달리한다.
- 포인터 핑은 hit testing을 가로막지 않으며 마크, 실행 취소 이력, 전체 지우기 대상, 저장 상태에 포함되지 않는다.
- 동작 줄이기 설정에서는 한 개의 정적 강조가 `150ms` 안에 사라지며 공간 이동을 사용하지 않는다.

## 마크 삭제 도구와 이력

- 마크 삭제 도구는 플로팅 마커의 1층 버튼으로 진입한다. 마크 하나를 지워도 도구가 유지되며 다른 도구를 고를 때만 종료된다.
- 그리기 모드에서 로컬 키 `E`를 누르면 패널을 열지 않고 마크 삭제 도구를 활성화한다. `E`는 텍스트 단축키로 재설정할 수 없고, 텍스트 편집 중에는 글자 입력이 우선한다.
- 포인터를 마크 위에 올리면 실제 hit 영역을 기준으로 지워질 마크 하나를 강조한다. 겹친 곳에서는 가장 위의 마크만 대상이다.
- 마크를 한 번 클릭하면 마크 전체를 제거한다. 빈 곳 클릭은 아무 변경도 만들지 않는다.
- 삭제 한 번은 실행 취소 한 단위다. `⌘Z`는 삭제한 마크를 원래 z-order에 복원하고 `⇧⌘Z`는 다시 삭제하며, 두 경우 모두 마크 삭제 도구를 유지한다.
- 부분 지우기와 드래그로 여러 마크를 쓸어 지우는 동작은 제공하지 않는다.

## 크기 단축키

- `⌘+`와 `⌘−`는 현재 도구가 사용하는 크기를 한 단계씩 증감한다.
- 자유곡선과 빠른 추가에서는 현재 펜 굵기를 바꾸고, 텍스트 도구 또는 텍스트 편집 세션에서는 텍스트 크기를 바꾼다.
- 마크 삭제 도구에서는 크기 단축키가 아무 동작도 하지 않는다.
- 최소·최대 단계에서는 현재 값을 유지한다. 속성 패널과 단축키는 같은 5단계 값을 사용한다.

## 상태 우선순위

1. 텍스트 편집 세션은 타이핑과 텍스트 크기 변경을 우선한다.
2. `⌘`으로 시작한 이동 가능한 펜·텍스트 마크의 이동 제스처는 현재 도구보다 우선한다.
3. 마크 삭제 도구는 클릭 삭제를 소유하며 핑·그리기를 만들지 않는다.
4. 빠른 추가는 한 번의 드래그를 소유한다.
5. 텍스트 도구는 한 번의 클릭으로 새 텍스트를 시작한다.
6. 나머지 입력은 자유곡선과 더블클릭 판정으로 처리한다.

## 인수 조건

- 플로팅 마커에 자유곡선·텍스트·마크 삭제가 같은 도구 위계로 보이고 블랙보드는 구분선 뒤에 보인다.
- 자유곡선 속성 패널 첫 줄에서 자유선과 네 빠른 추가를 확정 순서로 고를 수 있고, 빠른 추가 마크 하나를 확정한 뒤 자유곡선으로 돌아간다.
- 사각형·타원·삼각형·끝점 화살표는 빠른 추가로, 직선은 홀드 보정으로 현재 잉크 속성에 맞춰 생성되며 각각 실행 취소 한 단위다.
- 홀드 보정은 `350ms`에 사각형·타원·삼각형·직선만 미리보기하고, 움직이면 취소되며, 화살표를 자동 생성하지 않는다.
- `T` 뒤 클릭은 새 텍스트를 만들고, 텍스트 더블클릭은 교정하며, 그 밖의 더블클릭은 포인터 핑을 만든다.
- 포인터 핑을 빠르게 세 번 입력하면 독립된 버스트 세 개가 겹쳐 보이고 각각 자신의 `500ms` 수명이 끝날 때 제거된다.
- 마크 삭제 도구로 여러 마크를 연속 삭제할 수 있고, 삭제·실행 취소·다시 실행 뒤에도 삭제 도구가 유지된다.
- `E`로 마크 삭제 도구에 진입할 수 있고 텍스트 편집 중의 `E`는 도구를 바꾸지 않는다.
- `⌘+`와 `⌘−`가 자유곡선·빠른 추가의 굵기와 텍스트의 크기를 같은 단계 규칙으로 바꾼다.
- 새 동작이 펜·텍스트 마크 이동, 전체 지우기, 블랙보드, 두 단계 Esc와 통과 모드를 회귀시키지 않는다.

## 비범위

- 확정된 도형·직선·화살표의 이동, 크기, 회전, 꼭짓점, 끝점 또는 화살촉 교정
- 시작점·양방향 화살촉, 연결선, 스냅·정렬
- 지속적인 선택 상태, 크기 조절 핸들, 여러 마크 선택
- 부분 지우기, 드래그 연속 삭제, 삭제 크기 조절
- 포인터 핑의 색·반경·속도 설정과 지속적인 커서 하이라이트·스포트라이트
- 화살표 자동 인식 또는 자동 생성

## 가정과 남은 위험

- 빠른 추가 삼각형은 항상 위를 향하는 이등변 삼각형으로 시작한다. 회전 요구가 실제 사용에서 확인되기 전에는 방향 옵션을 추가하지 않는다.
- 삼각형 홀드 보정은 손그림 편차와 사각형 오분류를 함께 다뤄야 하므로 실제 마우스 표본으로 분류 임계값을 검증해야 한다.
- 더블클릭 첫 클릭의 점 마크를 사후 회수하는 동안 짧은 노출이 생길 수 있다.
- Web Animations 기반 독립 버스트와 실제 non-activating `NSPanel`의 빠른 더블클릭 전달은 네이티브 실행으로 확인해야 한다.
