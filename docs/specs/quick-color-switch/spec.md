# 빠른 색 전환

## 목적

판서 흐름을 끊지 않고 키 입력 한 번으로 잉크 색을 바꾼다. 색을 바꾸기 위해 플로팅 마커까지 포인터를 옮기고 패널을 여닫는 왕복을 없앤다. 텍스트 편집 세션 중에도 같은 키가 동작한다.

## 색 단축키

- 그리기 모드의 `⌘1`–`⌘5`는 [도구 속성 패널](../floating-marker-control/spec.md)의 색 순서 그대로 옐로우·오렌지·핑크·그린·블루를 즉시 선택한다.
- `⌘` 외의 수정자가 섞인 조합은 색 단축키가 아니다. 수정자 없는 단독 숫자는 색을 바꾸지 않는다.
- 물리 키 기준으로 판정해 한글 입력 소스에서도 동작하며, 숫자패드 `⌘1`–`⌘5`도 같은 색을 선택한다.
- 색 변경은 현재 도구와 잠금 상태를 바꾸지 않는다. 자유곡선·기하·텍스트·마크 삭제 어떤 도구가 활성이어도 이후 만들 마크의 잉크 속성에만 적용된다.
- 점 커서, 플로팅 마커의 현재 도구 아이콘, 열린 도구 속성 패널의 색 선택 상태가 즉시 새 색을 반영한다.
- `⌘`를 잠깐 눌러 시작되는 이동 탐색은 숫자까지 이어지면 굵기 단축키 `⌘+`·`⌘−`와 같은 코드 억제 계약을 따른다.

## 텍스트 편집 세션과의 관계

- 텍스트 편집 세션 중에도 `⌘1`–`⌘5`는 색을 바꾼다. 굵기·텍스트 크기의 `⌘+`·`⌘−`와 같은 우선순위 계약이다.
- 새 텍스트 초안은 현재 잉크 색을 따르므로 편집 중 색 전환이 초안과 캐럿에 즉시 반영되고 그 색으로 확정된다.
- 기존 텍스트 마크의 교정 세션은 마크의 원래 색을 보존한다. 교정은 내용·크기만 바꾼다는 [텍스트 마크 교정](../text-mark-editing/spec.md) 계약을 따르며, 이때 `⌘1`–`⌘5`는 잉크 색만 바꾼다.
- 수정자 없는 `1`–`5`는 편집 세션에서 글자로 입력된다.

## 우선순위와 경계

- 포인터를 누르고 있는 동안(진행 중 획·기하 미리보기·이동/삭제 제스처)에는 색 단축키를 무시한다. 색은 포인터 제스처를 시작할 때 결정한다는 [잉크 속성 계약](../drawing/spec.md)을 따른다.
- 열린 도구 속성 패널이 있으면 색을 적용하고 패널을 닫는다. 패널에서 색을 고를 때와 같은 계약이다.
- 발견성: 도구 속성 패널 색 선택지의 접근성 이름과 툴팁에 해당 단축키를 병기한다.

## 인수 조건

- 그리기 모드에서 `⌘3`을 누르면 즉시 핑크가 되고 점 커서와 플로팅 마커 아이콘이 이를 반영한다.
- 새 텍스트 초안 입력 중 `⌘3`을 누르면 초안이 즉시 핑크로 보이고 그대로 확정되며, 수정자 없는 `3`은 글자로 입력된다.
- 기존 텍스트 마크 교정 중 `⌘3`을 눌러도 그 마크의 색은 유지되고, 교정을 마친 뒤 만든 새 마크는 핑크다.
- 기하 그리기 도구 또는 잠긴 마크 삭제 도구가 활성인 상태에서 색 단축키를 눌러도 도구와 잠금 상태가 유지된다.
- 획을 그리는 도중 누른 색 단축키는 진행 중 획과 그 확정 마크에 적용되지 않으며, 포인터를 놓은 뒤 다시 누르면 적용된다.
- 도구 속성 패널이 열린 상태에서 색 단축키를 누르면 색이 적용되고 패널이 닫힌다.
- `⌘`를 눌러 이동 탐색이 드러난 뒤 숫자로 이어지면 색만 바뀌고 이동 제스처가 시작되지 않는다.

## 비범위

- 색 단축키의 커스터마이즈와 설정 UI 노출 — 로컬 키 `E`와 같이 고정 키다.
- 기존 텍스트 마크 교정 중 마크 재색칠 — 교정 계약(내용·크기)의 확장은 별도 결정이다.
- 순환 키, 커서 옆 임시 팔레트 등 다른 빠른 전환 방식
- 굵기·텍스트 크기의 숫자 키 직접 선택
- 5색 팔레트 자체의 변경

## 남은 위험

- 수정자 선택(`⌘`)은 `⌥`(삭제 탐색·특수문자 입력 충돌)과 `⌃`(macOS Spaces 전환 충돌)를 배제한 소거로 정했다. 실제 사용에서 `⌘` 홀드 이동 탐색과의 체감 간섭은 macOS에서 확인한다.
- 향후 `⌘숫자`를 쓰는 다른 기능이 생기면 키 공간이 충돌한다. `⌘숫자`의 첫 소유자는 색 선택이다.
