# 빠른 색 전환

## 목적

판서 흐름을 끊지 않고 키 입력 한 번으로 잉크 색을 바꾼다. 색을 바꾸기 위해 플로팅 마커까지 포인터를 옮기고 패널을 여닫는 왕복을 없앤다.

## 색 단축키

- 그리기 모드의 로컬 키 `1`–`5`는 [도구 속성 패널](../floating-marker-control/spec.md)의 색 순서 그대로 옐로우·오렌지·핑크·그린·블루를 즉시 선택한다.
- 수정자 없는 단독 입력만 색을 바꾼다. `⌘`·`⌥`·`⌃`·`⇧` 조합은 색 단축키가 아니다.
- 물리 키 기준으로 판정해 한글 입력 소스에서도 동작하며, 숫자패드 `1`–`5`도 같은 색을 선택한다.
- 색 변경은 현재 도구와 잠금 상태를 바꾸지 않는다. 자유곡선·기하·텍스트·마크 삭제 어떤 도구가 활성이어도 이후 만들 마크의 잉크 속성에만 적용된다.
- 점 커서, 플로팅 마커의 현재 도구 아이콘, 열린 도구 속성 패널의 색 선택 상태가 즉시 새 색을 반영한다.

## 우선순위와 경계

- 텍스트 편집 세션 중에는 글자 입력이 우선한다. `1`–`5`는 텍스트 초안에 숫자로 입력되고 색은 바뀌지 않는다. 로컬 키 `E`와 같은 계약이다 — [포인터 핑과 마크 삭제 보조 진입](../quick-mark-tools/spec.md) 참조.
- 텍스트 도구가 활성이지만 편집 세션이 없으면 색 단축키가 동작하고 다음 텍스트 마크의 색에 적용된다.
- 포인터를 누르고 있는 동안(진행 중 획·기하 미리보기·이동/삭제 제스처)에는 색 단축키를 무시한다. 색은 포인터 제스처를 시작할 때 결정한다는 [잉크 속성 계약](../drawing/spec.md)을 따른다.
- 열린 도구 속성 패널이 있으면 색을 적용하고 패널을 닫는다. 패널에서 색을 고를 때와 같은 계약이다.
- 발견성: 도구 속성 패널 색 선택지의 접근성 이름과 툴팁에 해당 숫자 키를 병기한다.

## 인수 조건

- 그리기 모드에서 `3`을 누르면 즉시 핑크가 되고 점 커서와 플로팅 마커 아이콘이 이를 반영한다.
- 텍스트 편집 세션 중 `3`은 글자 `3`으로 입력되고 잉크 색은 유지된다.
- 기하 그리기 도구 또는 잠긴 마크 삭제 도구가 활성인 상태에서 색 단축키를 눌러도 도구와 잠금 상태가 유지된다.
- 획을 그리는 도중 누른 색 단축키는 진행 중 획과 그 확정 마크에 적용되지 않으며, 포인터를 놓은 뒤 다시 누르면 적용된다.
- 도구 속성 패널이 열린 상태에서 색 단축키를 누르면 색이 적용되고 패널이 닫힌다.
- 숫자패드 `1`–`5`가 상단 숫자 키와 같은 색을 선택한다.

## 비범위

- 색 단축키의 커스터마이즈와 설정 UI 노출 — 로컬 키 `E`와 같이 고정 키다.
- 순환 키, 커서 옆 임시 팔레트 등 다른 빠른 전환 방식
- 굵기·텍스트 크기의 숫자 키 직접 선택
- 5색 팔레트 자체의 변경

## 남은 위험

- 방식 선택(숫자 키 직접 선택)은 확인 문답이 닫혀 가정으로 채택했다. 순환 키나 커서 옆 팔레트를 선호하면 구현 전에 이 스펙을 재논의한다.
- 향후 로컬 숫자 키를 쓰는 다른 기능이 생기면 키 공간이 충돌한다. 숫자 키의 첫 소유자는 색 선택이다.
