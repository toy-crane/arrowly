# 텍스트 마크 작성과 교정

## 목적

마우스 손글씨 대신 키보드로 청중이 읽을 수 있는 짧은 텍스트를 강의 흐름 안에서 추가한다. 텍스트 마크만 내용, 텍스트 크기, 위치를 국소 교정할 수 있으며 일반적인 선택·편집 모델로 확장하지 않는다.

## 텍스트 도구와 편집 세션

- 텍스트 도구는 새 텍스트 초안의 위치를 정하거나 기존 텍스트 마크를 교정할 때 사용한다.
- 텍스트 도구를 선택한 상태와 텍스트 편집 세션은 서로 다르다. 도구만 선택한 상태에서는 아직 초안이 없다.
- 기본 텍스트 키를 누른 뒤 원하는 위치를 클릭하거나 빈 곳을 더블클릭하면 새 편집 세션을 시작한다.
- 기존 텍스트 마크는 현재 도구와 관계없이 더블클릭하면 편집한다. 텍스트 도구가 선택돼 있으면 한 번 클릭으로 편집한다.
- 새 텍스트는 현재 잉크 색을 사용한다. 기존 텍스트를 교정할 때는 원래 색을 유지한다.

## 확정과 종료

- Enter 또는 편집 영역 바깥 클릭은 현재 내용을 확정하고 편집 세션을 끝낸다.
- IME 조합 중 Enter는 편집 확정으로 처리하지 않는다.
- 편집 중 첫 Esc는 현재 내용을 확정하고 편집 세션만 끝낸다. 그리기 모드는 유지한다.
- 편집 세션이 없는 다음 Esc는 통과 모드로 나간다.
- 새 텍스트 입력 중 ⌘Z는 초안 전체를 취소하고 그리기 모드에 남는다.
- 기존 텍스트를 공백과 줄바꿈뿐인 값으로 확정하면 해당 마크를 삭제한다.

## 줄바꿈과 배치

- Shift+Enter로만 수동 줄바꿈한다. 자동 줄바꿈과 고정 폭은 사용하지 않는다.
- 빈 줄과 붙여넣은 줄바꿈을 보존한다.
- 줄은 좌측 정렬하고 줄간격은 글자 크기의 1.2배다.
- 편집 영역은 가장 긴 줄과 전체 줄 수에 맞춰 자동으로 확장한다.
- 캔버스 표시, 줄별 hit testing, 클릭한 위치의 캐럿 계산은 같은 텍스트 배치를 사용한다.
- 캐럿은 클릭한 줄에서 가장 가까운 grapheme 경계에 놓인다.

## 텍스트 크기

- 텍스트 크기는 펜 굵기와 독립된 `24/32/44/60/80px` 다섯 단계다.
- 첫 기본값은 `32px`다.
- 편집 중 ⌘+/⌘− 또는 플로팅 마커의 도구 속성 패널로 한 단계씩 바꾼다. 최대·최소에서는 현재 값을 유지한다.
- 새 텍스트 마크를 확정하면 그 크기가 다음 새 텍스트의 기본값이 된다.
- 새 초안을 취소하거나 기존 텍스트 마크를 교정할 때는 다음 기본값을 바꾸지 않는다.
- 편집 세션이 없을 때 플로팅 마커에서 고른 크기는 즉시 다음 기본값으로 저장한다.
- 서체는 시스템 산세리프를 사용해 손그림과 구분되는 정확한 정보로 보이게 한다.

## 텍스트 이동 제스처

- 텍스트 도구로 기존 텍스트 마크를 누르면 클릭 후보가 된다.
- 포인터 이동이 `4px` 이내면 놓을 때 편집을 시작하고, `4px`을 넘으면 텍스트 이동 제스처로 전환한다.
- 이동은 잡은 지점과 텍스트 좌상단의 상대 위치를 유지하고 새 위치를 실시간으로 보여준다.
- 포인터를 놓으면 위치 변경을 실행 취소 한 단위로 기록하고 자유곡선 도구로 돌아간다.
- 포인터 입력이 취소되면 원래 위치를 복원하고 텍스트 도구를 유지한다.
- 화면 경계 제한이나 스냅은 적용하지 않는다.

## 이력

- 새 텍스트 추가, 내용·크기 교정, 이동, 삭제는 각각 실행 취소 한 단위다.
- 값이 바뀌지 않은 편집은 이력을 만들지 않는다.
- Undo/Redo는 겹친 텍스트의 원래 z-order를 보존한다.
- 겹친 텍스트를 편집할 때는 가장 위에 있는 마크를 선택한다.

## 자동 인수 조건

- 여러 줄 한글, 이모지, 빈 줄이 편집 영역과 캔버스에서 같은 위치와 줄간격으로 보인다.
- 여러 줄의 각 줄을 클릭하면 가장 가까운 grapheme 경계에서 편집을 시작한다.
- `4px` 이하 클릭과 `4px` 초과 끌기가 편집과 이동으로 안정적으로 분리된다.
- 이동 미리보기에는 원본 텍스트가 중복 표시되지 않는다.
- 확정, 취소, 삭제, 이동의 Undo/Redo가 원래 값과 z-order를 복원한다.
- 펜 입력, 전체 지우기, 직선 잠금, 두 단계 Esc가 회귀하지 않는다.

## 네이티브 인수 조건

다음은 실제 macOS 오버레이에서 확인한다.

1. 한글 2벌식 조합 중 밑줄과 후보창이 보이고 조합 중 Enter가 세션을 끝내지 않는다.
2. 빈 줄을 포함한 Shift+Enter 줄바꿈이 확정 뒤에도 같은 모양으로 보인다.
3. 기존 여러 줄 텍스트를 더블클릭하면 클릭한 줄과 글자 근처에 캐럿이 놓인다.
4. 텍스트 도구에서 실제 마우스 클릭과 끌기가 각각 편집과 이동으로 구분된다.
5. 블랙보드에서도 작성, 교정, 이동, Undo/Redo가 동일하게 동작한다.
6. 웹뷰가 응답할 때 첫 Esc는 편집만 끝내고, 웹뷰가 멈춘 뒤 다음 Esc는 통과 모드로 나간다.

## 비범위

- 자동 줄바꿈과 고정 폭·크기 조절 가능한 텍스트 박스
- 텍스트 정렬, 선택 상태, 이동 핸들
- 여러 텍스트 또는 여러 마크 동시 이동
- 텍스트 외 마크의 이동·교정
