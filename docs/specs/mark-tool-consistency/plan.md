# Approach

기존 캔버스 구조를 유지하면서 이동·삭제 상호작용만 작은 순수 상태 정책으로 분리한다.

- 마크 모델과 이력 저장소는 유지한다.
- “빠른 추가”를 지속형 기하 그리기 도구로 정리하고 자동 도형 분류와 Shift 방향 제한을 제거한다.
- 후보 표시 영역과 실제 hit-test가 어긋나지 않도록 shared drawing에 공통 표적화 계약을 둔다.
- 이동·삭제의 `대기 → 전체 탐색 → 직접 대상 → 제스처 종료` 전이는 순수 interaction policy가 소유한다.
- 캔버스는 포인터·키보드 이벤트 전달, 마크 이동·삭제, 공통 후보 렌더링만 담당한다.
- 플로팅 마커는 현재 도구·색·굵기를 표현하고 도구 선택 상태를 전달한다.

기각한 대안:

- 캔버스 내부에 boolean과 타이머만 추가: 현재도 복잡한 상태 조합을 더 취약하게 만든다.
- 전체 캔버스 입력을 한 번에 reducer로 재작성: 텍스트·IME·더블클릭 회귀 범위가 지나치게 넓다.
- 일반 선택 레이어 도입: 지속 선택과 객체 편집으로 제품 경계를 넘는다.

# Order

첫 구현 세션에서는 순수 도메인과 도구 동작을 정리한다. 자동 보정과 방향 제한을 제거하고, 기하 도구 생성·지속 상태·아이콘 굵기·모든 마크의 공통 hit-test를 먼저 완성한다. 이 단계가 끝나면 shared drawing과 플로팅 마커 테스트만으로 도구 및 표적화 계약을 검증할 수 있다.

두 번째 세션에서는 이동·삭제 interaction policy와 공통 탐색 화면을 캔버스에 연결한다. `⌘`·`⌥`의 200ms 대기, 즉시 제스처, 4px 임계값, 잠금형 `E`, 취소·포커스 상실·Undo/Redo를 통합한다. 기존 텍스트 편집과 포인터 핑은 이 정책의 우선순위 바깥에서 그대로 유지한다.

마지막 검증 패스에서는 전체 자동 게이트와 실제 macOS 오버레이를 확인한다. 구현이 스펙과 다르면 코드를 고치고, 결정 자체가 달라져야 하면 스펙으로 되돌린다. 완료 후 이 계획은 삭제하고 Git 이력만 남긴다.

# Acceptance criteria

- [ ] 도구 패널이 `자유선 → 직선 → 화살표 → 사각형 → 타원 → 삼각형` 순서다.
- [ ] 기하 도구로 두 개 이상 연속 생성해도 도구가 유지되고, 현재 도구 버튼 재클릭은 패널만 토글한다.
- [ ] 접힌 기하 도구 아이콘이 현재 색·굵기를 반영하고 패널 아이콘은 고정 굵기다.
- [ ] 자유곡선의 진행 링과 자동 도형·직선 변환이 완전히 사라진다.
- [ ] Shift 유무와 관계없이 직선·화살표 끝점이 포인터 위치와 일치한다.
- [ ] 펜·텍스트·직선·화살표·닫힌 도형 모두 이동과 삭제가 가능하다.
- [ ] 후보로 표시되는 전체 영역과 실제 hit 영역이 같고, 겹치면 최상단 마크가 선택된다.
- [ ] 멀티라인 텍스트의 빈 줄과 짧은 줄 오른쪽도 이동·삭제 대상으로 잡힌다.
- [ ] `⌘`·`⌥` 전체 후보 화면은 200ms에 나타나고, 그 전 직접 드래그·클릭은 즉시 동작한다.
- [ ] 4px 이하 Command 제스처는 아무 마크나 이력을 만들지 않는다.
- [ ] Option 삭제는 같은 대상에서 pointer-up 할 때 확정되며 취소·바깥 release는 삭제하지 않는다.
- [ ] `E`와 삭제 버튼은 같은 탐색 화면을 잠그고 `E` 재입력이나 다른 도구 선택으로 종료한다.
- [ ] 이동·삭제·취소·Undo/Redo 후 원래 z-order와 선택 도구가 보존된다.
- [ ] 그리기 모드 종료 시 자유곡선으로 초기화되고, 전체 지우기는 현재 기하 도구를 유지한다.
- [ ] 텍스트 편집, IME, Shift+Enter, `⌘Z`, `⇧⌘Z`, `⌘+`, `⌘−`, `⌥⌫`, `⌥Tab`, 포인터 핑이 회귀하지 않는다.

# Verification and seams

- 기하 생성 단위 테스트: 드래그 방향, 끝점, 화살촉, 4px 미만 입력.
- shared drawing 단위 테스트: 모든 마크의 공통 hit 영역, 후보 프레임, 최상단 우선, 평행 이동과 이력.
- interaction policy 단위 테스트: 199/200ms 경계, 단축키 chord 억제, modifier 선행 조건, 취소와 modifier 조기 해제.
- 플로팅 마커 컴포넌트 테스트: 도구 지속, 패널 토글, 선택 순서, 현재 굵기와 중립 아이콘.
- 캔버스 상호작용 테스트: 실제 이벤트 순서, 공통 딤·후보 렌더링, pointer capture, 이동·삭제·Undo/Redo.
- 앱 통합 테스트: 모드 종료 초기화, 도구·커서·마커 상태 동기화.
- 전체 게이트: `bun run test:all`.
- 네이티브 검증: `bun run tauri dev`에서 실제 `NSPanel`로 `⌘`·`⌥`, 200ms, 아래 앱 비활성 유지, 밝은 화면·어두운 화면·블랙보드, IME를 확인한다.

# Off-limits

- Rust IPC, 전역 단축키 등록과 설정 스키마
- 마크 저장 형식과 현재 이력 모델의 전면 재작성
- 텍스트 내용·크기 편집 계약과 캐럿 hit-test
- 플로팅 마커 위치 저장 및 캡슐 외형
- 포인터 핑의 모양과 수명
- 지속 선택, 핸들, 크기 변경, 회전, 다중 선택
- 도구 선택의 앱 재시작 간 영속화

# Risks and open questions

- 넓어진 텍스트·도형 프레임이 뒤쪽 마크를 과도하게 가릴 수 있으므로 겹침 사례를 별도로 검증한다.
- 텍스트 편집용 줄별 hit-test와 이동·삭제용 전체 프레임 hit-test를 섞지 않는다.
- 현재 삭제는 pointer-down 즉시 확정되므로 pointer-up 확정으로 바꾸면서 더블클릭·포인터 핑과 충돌할 가능성이 있다.
- fake timer와 동기 `requestAnimationFrame` 스텁을 함께 쓰는 테스트는 예약 ID 회귀에 주의한다.
- non-activating `NSPanel`의 Option 이벤트와 modifier key-up 전달은 브라우저 테스트만으로 증명할 수 없다.
- 200ms 체감과 후보 색 대비는 네이티브 확인 후 결정 변경이 필요하면 스펙으로 되돌린다.
- 미해결 제품 결정은 없다.
