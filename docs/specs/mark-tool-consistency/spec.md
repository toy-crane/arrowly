# 마크 도구 선택·이동·삭제 일관성

## 목적

사용자가 선택한 그리기 도구와 잉크 굵기를 플로팅 마커에서 정확히 확인하고, 같은 마크를 연속해서 만들며, 모든 확정 마크를 일관된 탐색 화면에서 안정적으로 이동하거나 삭제한다. 자유곡선은 사용자의 획을 임의로 해석하지 않는다.

이 스펙은 기하 그리기 도구의 지속 상태, 플로팅 마커 표시, 마크 표적화, 이동·삭제 탐색 상태, 자동 보정 제거와 방향 제한 제거를 함께 소유한다.

## 도구 선택과 지속 상태

- 자유곡선·직선·끝점 화살표·사각형·타원·삼각형은 각각 명시적으로 선택하는 그리기 도구다.
- 도구 속성 패널의 그리기 순서는 `자유선 → 직선 → 화살표 → 사각형 → 타원 → 삼각형`이다.
- 한 번의 그리기 모드 세션에서 직선·화살표·사각형·타원·삼각형을 선택하면 다른 도구를 명시적으로 고를 때까지 같은 도구가 유지된다.
- 마크 하나를 확정해도 자유곡선으로 자동 복귀하지 않는다.
- 현재 그리기 도구 버튼을 다시 누르면 도구를 바꾸지 않고 도구 속성 패널만 열거나 닫는다.
- 자유곡선으로 돌아가려면 도구 속성 패널에서 자유선을 명시적으로 선택한다.
- 전체 지우기는 확정 마크와 진행 중 입력을 비우되 현재 선택한 그리기 도구를 바꾸지 않는다.
- 그리기 모드를 종료하면 일시적인 입력과 탐색 상태를 끝내고, 다음 진입의 그리기 도구는 기존 계약대로 자유곡선으로 초기화한다.

## 플로팅 마커와 잉크 표시

- 접힌 플로팅 마커의 현재 그리기 도구 버튼은 선택한 도구의 아이콘, 현재 잉크색과 현재 굵기의 상대 단계를 함께 표시한다.
- 자유곡선뿐 아니라 직선·화살표·사각형·타원·삼각형 아이콘도 현재 굵기를 반영한다.
- 도구 속성 패널의 첫 줄 아이콘은 도구 종류를 비교하기 위한 중립색·고정 굵기로 표시한다. 현재 잉크의 미리보기 역할을 하지 않는다.
- 색과 굵기를 바꾸면 접힌 현재 도구 아이콘과 이후 생성하는 마크에 즉시 반영한다.
- 포인터 제스처가 시작된 뒤의 색·굵기 변경은 진행 중 마크에 소급하지 않는다.

## 자유곡선과 명시적 기하 도구

- 자유곡선은 포인터 경로를 펜 마크로 확정한다. 획의 모양이나 정지 시간을 분석해 다른 마크로 바꾸지 않는다.
- 자유곡선의 끝에서 멈춰도 진행 링, 분류 대기, 도형 미리보기 또는 직선 미리보기를 표시하지 않는다.
- 자유곡선을 오래 누르고 놓아도 원래 포인터 경로의 펜 마크로 확정한다.
- 직선·화살표·사각형·타원·삼각형은 해당 기하 그리기 도구로만 생성한다.
- 사각형과 타원은 포인터 시작점과 끝점의 경계 상자에 맞춘다. 삼각형은 같은 경계 상자 안에서 위쪽 꼭짓점과 아래쪽 밑변을 가진 이등변 삼각형으로 만든다.
- 직선과 화살표는 시작점과 현재 포인터 위치를 그대로 끝점으로 사용한다.
- 화살표는 끝점에만 화살촉을 표시한다.
- 기하 도구에서 포인터 이동이 `4px` 이내면 마크를 만들지 않고 현재 기하 도구를 유지한다.
- 기하 입력이 취소되면 미리보기를 버리고 현재 기하 도구를 유지한다.
- 기하 그리기 중 Shift를 눌러도 방향을 수평·수직 또는 45° 단위로 제한하지 않는다.
- 이 Shift 제거는 그리기 제스처에만 적용한다. `Shift+Enter` 줄바꿈과 `⇧⌘Z` Redo 등 다른 계약은 유지한다.

## 공통 마크 표적화

- 펜·텍스트·직선·화살표·사각형·타원·삼각형을 포함한 모든 확정 마크는 이동과 삭제의 대상이다.
- 진행 중 획과 진행 중 기하 마크 미리보기는 아직 마크가 아니므로 대상이 아니다.
- 탐색 상태에서 후보로 보이는 영역 전체는 실제 포인터 hit 영역이어야 한다.
- 펜·직선·화살표는 강조된 경로 전체와 그 주변의 포인터 안전 여백 안에서 hit할 수 있다.
- 텍스트와 닫힌 도형은 화면에 표시한 후보 프레임 전체에서 hit할 수 있다. 멀티라인 텍스트의 짧은 줄 오른쪽과 빈 줄을 포함한다.
- 포인터가 실제 대상 위에 오면 후보 강조를 더 강하게 표시하고 현재 동작에 맞는 커서를 보여준다.
- 여러 마크가 겹친 위치에서는 화면에서 가장 위에 그려진 마크 하나를 대상으로 한다.
- 이동과 삭제는 같은 hit-test 결과를 사용한다. 같은 위치에서 이동은 되지만 삭제가 안 되거나 그 반대인 영역을 만들지 않는다.

## 공통 탐색 화면

- 이동 탐색 상태와 삭제 탐색 상태는 같은 배경 딤, 후보 표시, hover 강조와 hit 영역을 사용한다.
- 탐색 화면은 배경의 대비를 낮추되 마크의 원래 색과 형태를 식별할 수 있어야 한다.
- 이동 대상은 흰색 계열의 강한 강조, 삭제 대상은 붉은색 계열의 강한 강조로 현재 의도를 구분한다.
- 후보와 hover 강조에는 반복 애니메이션을 사용하지 않는다.
- 텍스트 편집 세션이나 활성 포인터 제스처가 있으면 새 탐색 상태를 시작하지 않는다.
- 창 포커스 상실, 문서 비표시 또는 그리기 모드 종료 시 탐색 상태와 후보 강조를 즉시 정리한다.

## 이동 탐색과 제스처

- `⌘`을 다른 키 없이 누르면 이동 탐색을 준비하고, `200ms` 유지하면 배경 딤과 모든 확정 마크의 후보를 표시한다.
- `⌘Z`, `⌘+`, `⌘−`처럼 비수식키가 이어지는 단축키 조합은 이동 탐색 화면을 표시하지 않는다.
- `200ms` 전에 `⌘`을 누른 채 마크에서 포인터를 시작하면 전체 탐색 화면을 기다리지 않고 그 마크만 즉시 이동 후보로 강조한다.
- 포인터를 누르는 순간 `⌘`이 눌려 있어야 한다. 포인터를 먼저 누른 뒤 `⌘`을 추가해도 이동으로 전환하지 않는다.
- 후보에서 포인터 이동이 `4px` 이내면 놓을 때 아무 변경도 만들지 않는다. `4px`을 넘으면 마크 이동 제스처를 시작한다.
- `4px` 이내에서 포인터를 놓으면 직접 대상 강조를 정리하고 현재 도구 입력도 만들지 않는다.
- 이동이 시작된 뒤 `⌘`을 놓아도 포인터를 놓거나 입력이 취소될 때까지 같은 마크 이동을 계속한다.
- 포인터를 놓으면 위치 변경을 실행 취소 한 단위로 기록하고 이동 전의 그리기 도구를 유지한다.
- 입력이 취소되면 원래 위치를 복원하고 이력을 만들지 않는다.
- 화면 경계 제한, 정렬과 스냅은 적용하지 않는다.

## 삭제 탐색과 제스처

- `⌥`을 다른 키 없이 누르면 삭제 탐색을 준비하고, `200ms` 유지하면 배경 딤과 모든 확정 마크의 후보를 표시한다.
- `⌥⌫`, `⌥Tab`처럼 비수식키가 이어지는 단축키 조합은 삭제 탐색 화면을 표시하지 않는다.
- `200ms` 전에 `⌥`을 누른 채 마크를 클릭하면 전체 탐색 화면을 기다리지 않고 그 마크만 즉시 삭제 대상으로 강조한다.
- 포인터를 누르는 순간 `⌥`이 눌려 있어야 한다. 포인터를 먼저 누른 뒤 `⌥`을 추가해도 삭제로 전환하지 않는다.
- 삭제는 같은 대상 위에서 포인터를 놓을 때 확정한다. 포인터 입력이 취소되거나 누른 대상 밖으로 드래그하면 삭제하지 않는다.
- 삭제 대상을 누른 뒤 `⌥`을 먼저 놓아도 해당 포인터 입력은 같은 대상 위에서 놓거나 취소될 때까지 삭제 제스처를 유지한다.
- `⌥`을 계속 누른 채 다른 마크를 순서대로 클릭하면 여러 마크를 연속 삭제할 수 있다.
- `⌥`을 놓으면 삭제 탐색 상태를 끝내고 이전 그리기 도구로 돌아간다.
- 삭제 한 번은 실행 취소 한 단위다. `⌘Z`는 삭제한 마크를 원래 z-order에 복원하고 `⇧⌘Z`는 다시 삭제한다.
- 부분 지우기와 드래그로 여러 마크를 쓸어 지우는 동작은 제공하지 않는다.

## 삭제 도구의 보조 진입

- 플로팅 마커의 마크 삭제 버튼과 로컬 키 `E`는 포인터만 사용하는 사용자와 기능 발견을 위한 보조 진입으로 유지한다.
- 버튼이나 `E`로 진입하면 같은 삭제 탐색 화면을 잠금 상태로 표시하며, 다른 도구를 고르거나 `E`를 다시 누르면 이전 그리기 도구로 돌아간다.
- 잠금 상태에서도 hit 영역, 대상 강조, 삭제 확정, Undo/Redo 계약은 `⌥`의 순간 삭제 탐색과 동일하다.
- 텍스트 편집 중에는 글자 입력이 우선하며 `⌥`·`E`·삭제 버튼으로 편집 세션을 중단하지 않는다.

## 상태 우선순위

1. 텍스트 편집 세션은 타이핑, 줄바꿈과 텍스트 크기 변경을 우선한다.
2. 이미 시작한 포인터 제스처는 새 이동·삭제 탐색 진입보다 우선한다.
3. `⌘`으로 시작한 마크 이동 제스처가 활성 도구보다 우선한다.
4. `⌥` 또는 잠긴 삭제 도구로 시작한 삭제 제스처가 그리기 도구보다 우선한다.
5. 현재 선택된 자유곡선·기하·텍스트 도구가 나머지 포인터 입력을 처리한다.

## 이력과 도구 보존

- 새 마크 추가, 마크 이동, 마크 삭제와 텍스트 교정은 각각 실행 취소 한 단위다.
- Undo/Redo는 이동·삭제 전후의 위치와 원래 z-order를 보존한다.
- 이동·삭제와 그 Undo/Redo는 현재 선택한 그리기 도구를 바꾸지 않는다.
- 새 변경이 생기면 취소 후 남아 있던 Redo 이력은 폐기한다.

## 자동 인수 조건

- 도구 속성 패널에서 직선·화살표·사각형·타원·삼각형을 선택해 각각 두 개의 마크를 연속 생성할 수 있고 선택 도구가 유지된다.
- 현재 도구 버튼을 다시 눌러도 자유곡선으로 바뀌지 않고 패널만 열고 닫힌다.
- 접힌 현재 도구 아이콘이 도구·색·굵기를 반영하고, 속성 패널의 도구 아이콘은 고정 굵기를 유지한다.
- 자유곡선을 그리다 멈추거나 오래 눌러도 진행 링과 자동 보정이 나타나지 않고 펜 마크로 확정된다.
- Shift를 누른 채 직선·화살표를 그려도 같은 포인터 위치를 끝점으로 사용한다.
- `⌘` 또는 `⌥`을 `199ms` 유지하면 전체 탐색 화면이 보이지 않고 `200ms`에 보인다.
- `⌘Z`, `⌘+`, `⌘−`, `⌥⌫`, `⌥Tab`은 이동·삭제 탐색 화면을 표시하지 않고 원래 단축키 동작을 유지한다.
- `200ms` 전의 직접 `⌘` 드래그와 `⌥` 클릭은 해당 마크를 즉시 대상으로 삼는다.
- 모든 확정 마크가 이동·삭제 후보로 표시되고 같은 hit 영역으로 동작한다.
- 멀티라인 텍스트의 후보 프레임 안 어느 곳에서도 이동과 삭제 대상을 잡을 수 있다.
- 가는 펜 마크와 직선은 강조 경로 주변의 안전 여백에서 이동과 삭제 대상을 잡을 수 있다.
- 겹친 마크에서는 가장 위의 마크 하나만 이동하거나 삭제한다.
- `4px` 이하의 `⌘` 제스처는 이동 이력을 만들지 않고, 이를 넘은 드래그만 이동한다.
- 이동·삭제 확정, 취소와 Undo/Redo가 위치와 z-order를 복원하며 현재 그리기 도구를 유지한다.
- `E`와 삭제 버튼의 잠금 상태가 `⌥` 순간 삭제와 같은 딤·후보·hit-test를 사용하고 `E` 재입력으로 종료된다.
- 텍스트 편집 중 `Shift+Enter`, `⌘Z`, `⇧⌘Z`와 크기 단축키가 회귀하지 않는다.

## 네이티브 인수 조건

다음은 실제 macOS 주석 오버레이에서 확인한다.

1. `⌘`과 `⌥`을 각각 `200ms` 유지할 때 아래 앱을 활성화하지 않고 동일한 배경 딤과 후보 필드를 표시한다.
2. 빠른 직접 `⌘` 드래그와 `⌥` 클릭이 전체 탐색 화면의 지연 없이 대상 강조와 동작을 시작한다.
3. 밝은 화면, 어두운 화면과 블랙보드에서 후보, 이동 대상, 삭제 대상과 원래 마크를 구분할 수 있다.
4. 실제 마우스로 가는 펜·직선, 작은 텍스트와 겹친 마크를 반복 표적화해 시각 후보와 hit 결과가 일치한다.
5. 수식키를 놓거나 포인터 입력을 취소하거나 창 포커스가 사라졌을 때 탐색 화면, 대상 강조와 도구 상태가 계약대로 수렴한다.
6. 텍스트 편집, IME, `⌘Z`, `⇧⌘Z`, `⌘+`, `⌘−`, 전체 지우기와 통과 모드가 새 수식키 탐색에 가로막히지 않는다.

## 비범위

- 지속적인 선택 상태와 선택 핸들
- 여러 마크 동시 선택·이동·삭제, 그룹, 올가미, 정렬과 스냅
- 도형 크기 변경, 회전, 꼭짓점·끝점 또는 화살촉 교정
- 부분 지우기와 드래그 연속 삭제
- 자유곡선의 도형·직선·화살표 자동 인식 또는 자동 생성
- 직선·화살표의 수평·수직·45° 방향 제한

## 가정과 남은 위험

- `E`와 삭제 버튼은 기능 발견과 포인터 전용 사용을 위해 잠금형 보조 진입으로 유지한다. 실제 사용에서 순간 `⌥` 진입만으로 충분하다는 근거가 쌓이면 별도로 단순화할 수 있다.
- 기하 도구의 지속 범위는 활성 그리기 모드 세션으로 두고, 모드를 나갔다 다시 들어오면 기존처럼 자유곡선에서 시작한다. 세션 간 도구 보존 요구가 확인되면 설정 영속화와 함께 별도로 결정한다.
- `200ms`는 의도하지 않은 수식키 탭의 화면 번쩍임을 줄이면서 탐색을 답답하게 만들지 않는 초기값이다. 실제 강의 중 반복 입력으로 확인한다.
- `⌥` 포인터 이벤트, 수식키 해제와 non-activating `NSPanel`의 포커스 전달은 브라우저 테스트만으로 보장하지 않고 네이티브 인수 조건으로 확인한다.
- 넓어진 hit 영역은 겹친 마크의 최상단 우선 규칙과 함께 검증해 뒤쪽 마크를 지나치게 가리지 않는지 확인한다.
