# 제품 경계

## 목적

Arrowly는 화면 녹화 강의와 대면 강의에서 정지된 화면 위에 순간적으로 그리고 지우는 일회용 주석 오버레이다. 강의 흐름을 깨지 않는 것이 북극성이며, 되돌릴 수 없는 라이브 상황에서는 기능 수보다 안정성과 탈출 안전장치를 우선한다.

## 사용 맥락

- 기존 화면 위에 밑줄, 동그라미, 손그림 화살표를 얹어 강조한다.
- 블랙보드 또는 기존 화면 위에 자유곡선, 텍스트와 소수의 기하 마크로 간단한 구조를 설명한다.
- 저장할 마크를 만들지 않고 한 지점을 잠깐 가리킬 때 포인터 핑을 사용한다.
- 한 장면은 보통 1~2분 사용하고, 설명이 끝나면 전체 지우기로 비운 뒤 새로 시작한다.
- 강의 전에 판서를 준비하거나 지난 판서를 불러오지 않는다. 준비된 자료는 슬라이드가 담당한다.

## 정체성 경계

- 판서 결과는 펜 마크, 직선 마크, 도형 마크, 텍스트 마크다.
- 모든 확정 마크는 강의 중 배치 실수를 바로잡기 위해 위치를 국소 교정할 수 있다.
- 텍스트 마크만 내용과 텍스트 크기를 추가로 교정할 수 있다.
- 펜 경로와 직선·도형의 크기·끝점·기하는 확정 뒤 교정하지 않고 삭제와 다시 그리기로 바로잡는다.
- 위치 교정은 일회성 마크 이동 제스처로 끝나며 지속적인 선택 상태를 만들지 않는다.
- 기하 그리기 도구는 끝점 화살표·사각형·타원·삼각형을 반복해서 정확히 만들고, 일반적인 도형 편집 모델로 확장하지 않는다.
- 자유곡선은 사용자가 획 끝을 오래 눌러 요청할 때만 직선·사각형·타원·삼각형으로 보정한다.
- 포인터 핑은 마크가 아닌 일시적 강조이며 이력이나 저장 상태를 만들지 않는다.
- 마크를 저장·배치·정렬하는 문서 모델을 만들지 않는다.

## 설계 원칙

- 사용자가 강의 중 표현을 만드는 입력 비용을 최소화한다.
- 기능은 실사용에서 반복해서 드러난 필요가 있을 때만 추가한다.
- 마크 하나를 즉석에서 남기는 기능은 범위 안이고, 객체를 선택·관리하는 기능은 범위 밖이다.
- 화면 전체를 덮는 도구이므로 정상 동작보다 탈출 가능성을 우선한다.

## 비범위

- 여러 마크 선택·이동·크기 변경, 그룹, 정렬, 연결선
- 그림 저장·불러오기, 판서 사전 준비, 여러 페이지
- 펜 마크의 경로 교정, 직선·도형 마크의 끝점·크기·기하 교정, 부분 지우개
- 자유곡선에서 화살표 자동 인식·생성
- 화살표의 수평·수직·45° 방향 제한
- 시작점·양방향 화살표
- 통과 모드에서 마크를 계속 표시하는 동작
- 다중 모니터 동시 판서, 아래 화면 스크롤 추적, 자동 정렬
- 지속적인 커서 하이라이트·스포트라이트, 화면 확대

## 제약

- 마크는 화면 좌표에만 존재하므로 아래 앱이 스크롤되면 원래 대상과 어긋날 수 있다.
- 현재 작업은 커서가 있는 모니터 하나에만 유효하다.
- macOS 전용이며 v1은 자동 업데이트를 제공하지 않는다.

## 경계 시나리오

- 사용자가 확정 마크 하나를 잘못 놓아 바로 옮기는 것은 국소 교정이므로 범위 안이다.
- 펜 경로 또는 직선·도형의 크기나 형태를 바꾸려는 요구는 삭제 후 다시 그리는 방식으로 해결한다.
- 여러 마크를 함께 선택해 정렬하려는 요구는 문서 편집이므로 범위 밖이다.
- 통과 모드에서 아래 화면을 움직여 기존 마크가 어긋나면 마크를 재배치하지 않고 전체 지우기로 새 장면을 시작한다.
