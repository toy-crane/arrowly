# 수동 줄바꿈과 텍스트 이동

## 목적

강의 흐름을 멈추지 않고 텍스트를 여러 줄로 적고 잘못 놓은 텍스트 하나를 바로 옮긴다. Arrowly는 선택·정렬 기능이 있는 다이어그램 편집기로 확장하지 않는다.

## 확정 동작

- 텍스트는 고정 폭 박스 없이 Shift+Enter로만 줄바꿈한다. Enter는 편집을 확정하며 IME 조합 중 Enter는 확정으로 처리하지 않는다.
- 빈 줄과 붙여넣은 줄바꿈을 그대로 보존한다. 전체가 공백과 줄바꿈뿐일 때만 빈 값으로 판단한다.
- 줄은 좌측 정렬하고 줄간격은 글자 크기의 1.2배다. 편집 윤곽은 가장 긴 줄과 전체 줄 수에 맞춰 자동 확장한다.
- 캔버스 렌더링, 줄별 6px 여유 hit testing, 클릭 커서 위치는 같은 레이아웃을 사용한다. 커서는 클릭한 줄의 가장 가까운 grapheme 경계에 놓는다.
- T 모드에서 기존 텍스트를 누르면 클릭 후보가 된다. 4px 이내에서 놓으면 편집하고, 4px을 넘으면 이동으로 전환한다.
- 이동은 잡은 지점과 텍스트 좌상단의 간격을 보존하고 live canvas에 미리 보인다. 놓으면 위치 변경을 Undo 한 단위로 기록하고 펜 모드로 복귀한다.
- 이동 중 pointer cancel은 원래 위치를 복원하고 T 모드를 유지한다. 화면 경계 제한이나 스냅은 적용하지 않는다.
- 펜 모드의 그리기와 기존 텍스트 더블클릭 재편집은 바뀌지 않는다.

## 비범위

- 자동 줄바꿈과 고정 폭·리사이즈 가능한 텍스트 박스
- 텍스트 정렬, 선택 상태, 이동 핸들, 여러 마크 선택·이동
- 텍스트 외 마크의 이동·편집

## 인수 조건

- 여러 줄 한글·이모지·빈 줄이 DOM 에디터와 캔버스에서 같은 위치와 줄간격으로 보인다.
- 여러 줄의 각 줄을 클릭하면 해당 줄의 가장 가까운 grapheme 경계에서 편집을 시작한다.
- T 모드에서 4px 이하 클릭과 4px 초과 드래그가 각각 편집과 이동으로 안정적으로 분리된다.
- 이동 미리보기에는 원본이 중복 표시되지 않고, 확정·Undo·Redo·pointer cancel이 원래 z-order를 유지한다.
- 펜 입력, 더블클릭 재편집, 2단계 Esc, 전체 지우기 동작에 회귀가 없다.
