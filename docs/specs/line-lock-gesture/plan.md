## Approach

기존 화살표 마크를 직선 마크로 교체한다. 닫힌 획의 사각형·타원 분류는 유지하고, 유효한 열린 획은 시작점과 현재 끝점을 잇는 직선으로 잠근다. 캔버스는 고정된 닫힌 도형 미리보기와 조절 가능한 직선 잠금을 구분한다. 직선은 원시 끝점을 보존하고 순수 각도 계산을 통해 Shift 사용 시에만 45° 단위로 투영한다. 포인터가 멈춰 있어도 Shift 누름·해제가 즉시 반영되도록 keydown과 keyup을 모두 처리한다.

화살표 타입·렌더링·자동 분류는 제거한다. 홀드 전에 놓은 손그림 화살표는 자유곡선으로 남지만, `450ms` 홀드한 유효한 열린 획은 모양과 관계없이 직선 계약을 따른다. 마크는 메모리에만 있으므로 저장 형식이나 IPC 마이그레이션은 필요 없다.

거절한 대안: 별도 직선 도구는 모드와 마커 복잡도를 늘린다. 사용하지 않는 화살표 타입 존치는 제거된 기능의 분기와 테스트를 남긴다. 화살촉을 인식해 잠금을 막는 방식은 오인식 규칙을 추가해 홀드 제스처를 불명확하게 만든다.

## Order

한 구현 세션으로 진행한다. 먼저 마크 모델·렌더러·순수 분류기를 직선 계약으로 바꾸고, `450/150ms` 상수와 45° 투영 계산을 단위 테스트로 고정한다. 이 지점에서 화살표 타입과 화살촉 렌더링이 사라졌는지 검증한다. 그다음 캔버스 제스처를 직선 잠금 상태로 연결하고, 잠금 후 포인터 이동, Shift 전환, 최종 pointerup 좌표, 취소·모드 전환을 통합 테스트로 고정한다. 마지막으로 전체 프런트 테스트·의존 경계·빌드를 실행하고 실제 macOS 패널에서 입력 전달과 체감을 확인한다.

## Acceptance criteria

- `149ms`에는 진행 링이 없고 `150ms`부터 표시된다. `449ms`에는 자유곡선이며 `450ms`에 직선으로 잠긴다.
- 잠금 전 `3.5px`를 초과한 이동은 타이머를 다시 시작하고, `24px`보다 짧은 획은 잠기지 않는다.
- 잠금 후 시작점은 고정되고 끝점만 포인터를 따라가며, 큰 이동에도 자유곡선으로 돌아가지 않는다.
- Shift 누름·해제는 포인터 이동 없이도 45° 제한과 자유 각도를 즉시 전환하고, pointerup의 최종 좌표가 커밋된 직선에 반영된다.
- 사각형·타원 보정은 유지되고 화살표 마크·자동 분류·화살촉 렌더링·전용 도구는 존재하지 않는다.
- 홀드 전에 놓은 손그림 화살표는 자유곡선으로 남고, 홀드한 유효한 열린 획은 직선이 된다.
- 직선은 마크 하나로 커밋되고 Undo/Redo 한 번으로 제거·복원된다.
- Clear All, 모드 종료, pointer cancel은 후보 링과 잠긴 직선을 남기지 않는다.

## Verification and seams

- 순수 도형 분류에서 닫힘 판정, 열린 획의 첫 점·마지막 점, 최소 크기와 조정 상수를 검증한다.
- 순수 각도 계산에서 수평·수직·대각선·음수 각도와 거리 보존을 검증한다.
- 마크 렌더러에서 직선 선분 하나만 그리고 화살촉 선분을 만들지 않는지 검증한다.
- 캔버스 통합 seam에서 fake timer와 캔버스 spy로 타이밍·상태 전이·Shift·Undo·정리를 검증한다.
- 전체 자동 검증은 `bun run test:all`, 실제 입력은 macOS 네이티브 체크리스트로 확인한다.

## Off-limits

- Rust, Tauri 창 상태, IPC와 전역 단축키
- 플로팅 마커, 설정, 온보딩, 텍스트 편집
- 사각형·타원 분류 임계값
- 직선 확정 후 선택이나 끝점 재편집
- 화살표 전용 도구·수식키·결과 생성
- 홀드 수치 사용자 설정

## Risks and open questions

- non-activating 패널에서 Shift keyup이 안정적으로 전달되는지는 실제 앱 검증이 필요하다.
- 마지막 pointermove 없이 pointerup만 이동한 경우 release 좌표를 최종 반영해야 한다.
- `450ms/3.5px` 오잠금과 `150ms` 진행 링의 시각적 잡음은 실사용 튜닝 위험으로 남는다.
