# 포인터 핑은 스스로 대비를 갖는다

포인터 핑은 어두운 바깥 가장자리와 밝은 안쪽 가장자리를 함께 둘러 배경과 무관하게 읽히도록 한다. 확정된 마크에는 [외곽선을 두지 않고](../specs/drawing/spec.md), 밝은 배경에서 색이 약하면 사용자가 `⌘1`–`⌘5`로 다른 색을 고르게 한다. 핑에는 그 탈출구가 없다. 핑의 색은 [설정 대상이 아니고](../specs/quick-mark-tools/spec.md) 오버레이가 덮는 화면도 고를 수 없다.

색 하나로는 해결되지 않는다. WCAG 1.4.11은 그래픽 요소에 인접 색 대비 `3:1`을 요구하는데, 임의 배경 위에서는 인접 색을 알 수 없다. 비율을 보장하는 유일한 방법은 인접 색을 직접 들고 다니는 것이다. 옐로우 `#FFD400`은 흰 배경에서 `1.4:1`이었고, 핑크 `#FF2D95`로 바꿔도 `3.3:1`에 그치며 중간 회색 `#919191` 부근에서는 다시 무너진다. 한쪽 방향 외곽선만으로도 부족하다. 어두운 외곽선은 어두운 배경에서 아무 일도 하지 않는다.

같은 결론이 여러 곳에서 독립적으로 나온다. macOS 포인터는 흰 외곽선에 검은 채움이고, Windows 포커스 비주얼은 바깥 검정과 안쪽 흰색을 겹친다. Camtasia는 커서 채움색과 외곽선색을 따로 노출한다. AR 텍스트 가독성 연구와 자막 실무는 판·외곽선·그림자·없음 순으로 같은 순위를 낸다. 배경 픽셀을 읽어 적응하는 방식은 투명 웹뷰 오버레이에서 아래 앱을 볼 수 없어 불가능하다.

마크와 핑에 같은 외곽선 규칙을 적용하는 일관성보다, 사용자가 손댈 수 없는 요소가 항상 보이는 쪽을 택한다.
